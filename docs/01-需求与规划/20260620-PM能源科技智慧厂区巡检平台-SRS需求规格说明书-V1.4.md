# 二、项目概述

## 2.1 项目背景

随着能源行业的快速发展，厂区设备数量持续增加，设备安全管理面临巨大挑战。传统的人工巡检方式主要依赖纸质记录和口头汇报，存在以下问题：

1. **巡检记录不规范**：纸质记录容易丢失、字迹潦草、信息不完整，无法有效追溯历史巡检数据，给设备管理带来极大隐患。

2. **问题整改跟踪困难**：发现问题后，整改流程依赖人工传递，缺乏系统化跟踪机制，导致问题整改不及时、整改结果无法有效验证。

3. **巡检执行监督缺失**：管理人员无法实时掌握巡检人员的位置和任务执行情况，存在漏检、代检等风险，巡检质量难以保障。

4. **数据统计分析滞后**：巡检数据分散在纸质表单中，统计分析工作量大且效率低，无法及时发现设备异常规律和高风险区域。

5. **应急响应能力不足**：设备异常告警依赖人工发现和上报，缺乏实时监控和自动告警机制，可能导致安全事故的发生。

为解决上述问题，建设智慧厂区巡检平台，通过移动互联网、GPS定位、二维码识别等技术手段，实现巡检任务数字化管理、问题整改闭环跟踪、实时监控可视化展示，全面提升厂区设备安全管理水平。

## 2.2 项目建设目标

本项目旨在建设一套覆盖PC管理后台、移动端APP和数字大屏的综合性设备巡检管理系统，实现以下目标：

1. **规范巡检执行流程**：通过系统化的巡检计划制定、任务自动生成和移动端执行引导，确保巡检工作按规定路线、规定时间、规定标准执行，消除漏检和代检现象，提升巡检覆盖率至100%。

2. **实现问题整改闭环**：建立从问题发现、工单创建、审核指派、整改执行到复核验收的完整整改流程，确保每个问题都有人负责、有时限要求、有结果验证，整改完成率和及时率显著提升。

3. **提升实时监控能力**：通过数字大屏集成设备状态监控、视频监控、巡检人员实时轨迹等信息，为管理人员提供全面的厂区运行态势感知，实现异常情况的快速发现和响应。

4. **沉淀数据支撑决策**：系统自动积累巡检记录、设备异常、整改数据等，通过统计分析功能帮助管理人员识别高风险设备和区域，为设备维护计划和安全管理决策提供数据支撑。

5. **提升移动作业效率**：为巡检人员提供导航式移动巡检体验，支持扫码签到、检查清单填写、拍照记录、语音备注等功能，减少纸质工作量，提升现场作业效率。

## 2.3 项目建设范围

本项目建设范围包括：

**系统端**：
- PC端管理后台（Web应用）
- 移动端H5应用（支持微信/浏览器访问）
- 数字大屏（监控中心展示）

**功能模块**：
- PC端：工作台、数字大屏、设备管理、巡检管理、整改管理、基础设置、组织管理、权限管理、系统配置
- 移动端：工作台、巡检模块、整改模块、我的

**数据范围**：
- 设备档案数据：厂区所有设备的基本信息、技术参数、维护记录
- 巡检业务数据：巡检计划、任务、记录、轨迹数据
- 整改业务数据：整改工单及流程数据
- 组织权限数据：部门、人员、角色、权限配置

**不包含的功能**：
- 设备物联网实时数据采集（预留接口，本期不实现）
- 与ERP/EAM等外部系统的数据对接（预留接口，本期不实现）

## 3.5 功能详细设计

### 3.5.1 工作台

**（一）功能需求概述**

工作台是用户登录系统后的默认首页，集中展示与当前用户相关的待办事项、统计图表、通知公告和制度文档，帮助用户快速了解工作状态并处理待办任务，提升日常工作效率。

**（二）功能设计**

**待办事项功能**

- 展示当前用户所有待处理的整改相关事项：待审核整改工单、待执行整改任务、待复核整改结果
- 每类待办显示数量角标，数量为0时不显示角标
- 点击待办项直接跳转至对应处理页面，处理完成后返回工作台，待办数量自动更新

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 待办类型 | 标签 | 整改审核/整改执行/整改复核 |
| 待办标题 | 文本 | 工单编号或任务名称 |
| 关联设备 | 文本 | 关联的设备名称 |
| 截止时间 | 日期时间 | 任务或工单的截止时间 |
| 紧急程度 | 标签 | 紧急/普通，默认普通 |

**统计图表功能**

- 展示巡检完成率趋势、设备异常分布占比、本月整改工单统计等关键指标
- 数据每次进入工作台时自动刷新，支持手动点击刷新按钮更新
- 图表支持悬停查看具体数值；数据加载中显示等待提示

**通知公告功能**

- 展示最新发布的通知公告列表，按发布时间倒序排列，最多展示最新10条
- 点击查看详情，支持附件下载
- 点击"查看更多"可查看全部公告

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 公告标题 | 文本 | 最多100字 |
| 发布时间 | 日期时间 | - |

**制度文档功能**

- 展示系统内制度文档列表，支持在线预览（PDF格式）和下载

**业务规则**

- 待办事项按截止时间升序排列，距截止时间不足24小时的待办高亮显示，提示"即将超期"
- 通知公告按发布时间倒序排列，最多展示最新10条，点击"查看更多"可查看全部

**边界与异常**

- **无待办**：待办事项为空时，显示"暂无待办，继续保持"
- **图表加载失败**：统计图表加载失败时，显示"数据加载失败，请刷新重试"并提供刷新按钮

**（三）业务逻辑关联**

- 待办事项数据来源于巡检管理模块（巡检任务）和整改管理模块（整改工单）
- 统计图表数据来源于巡检任务执行记录和整改工单状态
- 通知公告由系统配置模块的通知公告功能发布管理
- 制度文档由系统配置模块的制度文档功能上传管理

### 3.5.2 数字大屏

**（一）功能需求概述**

数字大屏是面向管理人员的实时监控展示界面，集中呈现厂区设备状态、巡检人员实时轨迹、视频监控画面、巡检统计数据和异常告警信息，为管理层提供全面的厂区运行态势感知，支持快速发现和响应异常情况。

**（二）功能设计**

**厂区分布功能**

- 展示厂区分布地图，标注各厂区位置
- 厂区标记点显示厂区名称和今日巡检状态
- 支持点击厂区标记查看该厂区详细信息

| 展示字段 | 字段类型 | 说明 |
|---------|---------|------|
| 厂区名称 | 文本 | 厂区标识名称 |
| 厂区位置 | 地图坐标 | 经纬度坐标，在地图上标注 |
| 今日巡检任务数 | 数字 | 该厂区今日计划巡检任务总数 |
| 已完成任务数 | 数字 | 该厂区今日已完成巡检任务数 |
| 进行中任务数 | 数字 | 该厂区今日正在执行的巡检任务数 |
| 未开始任务数 | 数字 | 该厂区今日未开始的巡检任务数 |
| 异常设备数 | 数字 | 该厂区当前存在异常的设备数量 |
| 厂区状态 | 标签 | 正常/告警/严重 |

**标记点状态规则**

- 正常状态：表示无异常设备且巡检进度正常
- 告警状态：表示存在1-5个异常设备或巡检进度滞后
- 严重状态：表示存在5个以上异常设备或巡检严重滞后

**设备状态监控功能**

实时展示全厂区设备运行状态统计，帮助管理人员快速掌握设备整体健康度。

| 展示字段 | 字段类型 | 说明 |
|---------|---------|------|
| 设备总数 | 数字 | 系统内所有设备数量 |
| 停用设备数 | 数字 | 当前状态为停用的设备数量 |
| 维修中设备数 | 数字 | 当前状态为维修中的设备数量 |
| 异常设备数 | 数字 | 当前存在未处理整改工单或巡检异常记录的设备数量 |

**业务规则**

- 设备总数统计范围：系统内所有已录入的设备
- 停用设备判定：设备状态为"停用"
- 维修中设备判定：设备状态为"维修中"
- 异常设备判定：设备存在未处理的整改工单或巡检异常记录
- 数据刷新频率：每30秒自动刷新一次

**交互功能**

- 点击任意统计数据可查看对应设备明细列表
- 明细列表支持按设备名称搜索
- 明细列表支持按所属单位筛选
- 明细列表展示字段：设备名称、设备编号、所属单位、设备分类、设备状态、安装位置
- 点击设备总数时显示全部设备，点击停用/维修中/异常设备数时仅显示对应设备

**视频监控功能**

展示关键区域的实时视频监控画面。

| 展示字段 | 字段类型 | 说明 |

| 监控状态 | 标签 | 在线/离线 |

**交互规则**

- 支持切换监控通道，下拉选择其他监控点
- 支持单路视频全屏查看
- 离线监控点显示"设备离线"提示
- 视频监控需提前在系统配置中配置监控设备连接信息（海康/大华等品牌）

**巡检统计功能**

| 关联设备 | 文本 | 告警关联的设备名称 |
| 告警时间 | 日期时间 | 告警产生时间，格式：MM-DD HH:mm |
| 处理状态 | 标签 | 未处理/处理中/已处理 |

### 3.5.4 巡检管理

巡检管理模块负责巡检计划的制定和巡检任务的跟踪管理。业务主办通过配置周期性巡检计划，系统自动按周期生成巡检任务并推送给巡检员；管理人员可实时查看任务执行状态和巡检记录，确保巡检工作按计划执行。

#### 3.5.4.1 巡检计划

**（一）功能需求概述**

负责配置和管理周期性巡检计划，支持多厂区管理。支持按路线巡检或按巡检点巡检两种模式，可配置超时规则和巡检要求。已启用的计划不可编辑/删除，但支持调整巡检人员。

**（二）功能设计**

**查询/筛选功能**

| 筛选字段 | 字段类型 | 说明 |
|---------|---------|------|
| 计划名称 | 文本输入 | 支持模糊搜索 |
| 所属单位 | 层级选择 | 来源于组织管理中已启用的部门，支持多厂区 |
| 状态 | 下拉选择 | 启用/停用 |

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 计划名称 | 文本 | - |
| 所属单位 | 文本 | 显示完整组织路径（如"总公司 / 杭州厂区"） |
| 巡检方式 | 标签 | 按路线/按巡检点 |
| 巡检内容 | 文本 | 按路线时显示路线名称，按巡检点时显示巡检点名称 |
| 执行人员 | 文本 | 多人时显示人数，可查看全部姓名 |
| 执行周期 | 文本 | 每日/每周/每月 |
| 状态 | 开关 | 启用/停用，直接切换 |

- 列表为空时显示"暂无巡检计划"

**操作列规则**

- 已启用的计划：仅可"管理巡检人员"
- 已停用的计划：可"编辑"和"删除"

**新增/编辑功能**

新增/编辑为独立页面，分四个步骤依次填写，每步校验通过后方可进入下一步。

*第一步：基础信息*

| 字段名称 | 字段类型 | 长度限制 | 是否必填 | 默认值 | 说明 |
|---------|---------|---------|---------|-------|------|
| 计划名称 | 文本输入 | 2-50字 | 是 | - | 巡检计划名称 |
| 所属单位 | 层级选择 | - | 是 | - | 来源于组织管理中已启用的部门，选中后显示完整组织路径 |
| 备注 | 文本输入 | 最多200字 | 否 | - | 计划说明 |

*第二步：巡检内容*

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 巡检方式 | 单选 | 是 | 按路线巡检 / 按巡检点巡检 |
| 巡检路线 | 下拉选择 | 是（按路线时） | 来源于基础设置中已启用的巡检路线，选中后可预览路线包含的巡检点 |
| 巡检点 | 多选 | 是（按巡检点时） | 来源于基础设置中已启用的巡检点 |

- 切换巡检方式时自动清空另一种模式的已选数据

*第三步：巡检人员*

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 执行人员 | 多选 | 是 | 来源于组织管理中已启用的人员，显示姓名和所属部门，支持逐个移除已选人员 |

*第四步：巡检设置*

| 字段名称 | 字段类型 | 是否必填 | 默认值 | 说明 |
|---------|---------|---------|-------|------|
| 执行周期 | 下拉选择 | 是 | 每日 | 每日/每周/每月 |

- 类型为"设备"时，关联设备为必填，且只能选择启用状态的设备
- 签到方式为"GPS"时，GPS坐标为必填，格式校验：纬度范围-90~90，经度范围-180~180
- 同一部门下巡检点名称不可重复
- 停用的巡检点不可被新建的巡检路线引用
- 关联检查清单后，巡检员到达该点签到时，系统自动展示对应清单的所有检查项供逐项填写

- **空数据**：列表为空时，显示"暂无巡检点，点击新增按钮创建"
- **关联保护**：删除被路线引用的巡检点时，提示具体路线名称，不允许强制删除

**（三）业务逻辑关联**

- 依赖组织管理模块提供部门树数据

### 3.5.7 组织管理

| 负责人 | 下拉选择 | - | 否 | - | 来源于组织管理中已启用的人员 |

- 删除成功：列表刷新，提示"删除成功"

- 点击状态开关直接切换，无需确认

**（二）功能设计**

**待办事项**

展示当前用户名下尚未处理的事项，分为两类：待执行的巡检任务和待处理的整改工单。每条待办项显示事项名称、所属类型及截止时间，点击可直接跳转至对应任务或工单详情。

| 列名 | 字段类型 | 说明 |
|------|---------|------|
| 事项名称 | 文本 | 巡检任务名称或整改工单标题 |
| 类型 | 状态标签 | 巡检任务 / 整改工单 |
| 截止时间 | 日期时间 | 任务或工单的要求完成时间 |

待办事项列表为空时显示："暂无待办事项"。

**巡检任务执行（点击待办中的巡检任务进入）**

点击待办事项中的巡检任务，进入巡检任务执行流程。该流程支持查看任务关联的检查点，通过地图导航到达检查点后完成签到、填写检查清单、拍照记录，并在发现异常时发起整改工单，同时支持查看任务的巡检轨迹回放。

进入巡检任务后，展示任务基本信息：

| 列名 | 字段类型 | 说明 |
|------|---------|------|
| 任务名称 | 文本 | 巡检任务的名称 |
| 计划执行时间 | 日期时间 | 任务要求的执行时间段 |
| 巡检路线 | 文本 | 来源于巡检路线管理中已启用的路线名称 |
| 检查点数量 | 数值 | 该任务包含的检查点总数 |
| 任务状态 | 状态标签 | 待执行 / 进行中 / 已完成 / 已超期 |

**导航巡检执行**：进入状态为待执行或进行中的任务后，按以下环节依次完成：

1. **地图导航**：以地图形式展示当前任务的巡检路线及全部检查点位置，标注已完成和待完成的检查点。点击"导航"按钮启动地图导航引导用户前往该检查点。

2. **检查点签到**：到达检查点后，支持以下三种方式完成签到：

| 签到方式 | 说明 |
|---------|------|
| 扫码签到 | 扫描检查点处的二维码完成签到 |
| NFC 签到 | 感应检查点处的 NFC 标签完成签到 |
| GPS 签到 | 系统自动获取当前位置，在有效范围内完成签到 |

3. **填写检查清单**：签到成功后，展示该检查点的检查项列表，逐项填写。检查项支持以下四种类型：

| 检查项类型 | 填写方式 |
|-----------|---------|
| 正常/异常选择 | 单选，选择"正常"或"异常" |
| 数值输入 | 输入检测数值 |
| 文字描述 | 输入文字说明 |
| 拍照上传 | 拍摄现场照片并上传 |

4. **发起整改**：检查项选择"异常"后，支持针对该异常发起整改工单，填写异常描述和整改要求后提交，整改工单同步至整改工单管理。

**轨迹回放**：在已完成的任务详情中，支持查看该任务执行过程中的完整巡检轨迹，轨迹以路径形式在地图上展示，支持播放和暂停操作。进行中的任务支持查看实时轨迹，已完成任务展示完整轨迹。

**统计图表**

展示当前用户在当前统计周期内的巡检关键指标，包括巡检完成率和异常数量。巡检完成率以已完成任务数占应完成任务总数的百分比呈现；异常数量为当前周期内检查项被标记为异常的累计数量。

| 指标名称 | 字段类型 | 说明 |
|---------|---------|------|
| 巡检完成率 | 百分比 | 已完成任务数 / 应完成任务总数 |
| 异常数量 | 数值 | 当前周期内异常检查项的累计数量 |

**通知公告**

展示来源于系统公告管理中已发布的公告消息列表，按发布时间倒序排列。每条公告显示标题和发布时间，点击可查看公告正文。

| 列名 | 字段类型 | 说明 |
|------|---------|------|
| 公告标题 | 文本 | 公告的标题内容 |
| 发布时间 | 日期时间 | 公告的发布时间 |

列表为空时显示："暂无通知公告"。

**制度文档**

展示来源于制度文档管理中已发布的文档列表，按发布时间倒序排列。每条记录显示文档名称和发布时间，点击可查阅文档内容。

| 列名 | 字段类型 | 说明 |
|------|---------|------|
| 文档名称 | 文本 | 制度文档的名称 |
| 发布时间 | 日期时间 | 文档的发布时间 |

列表为空时显示："暂无制度文档"。

**业务规则**

- 待办事项仅展示分配给当前登录用户且状态为待执行的巡检任务，以及指派给当前用户且尚未处理的整改工单；其他用户的数据不展示。
- 超期任务在待办事项中高亮显示，并推送提醒："巡检任务【任务名称】已超期，请及时处理"。
- 统计图表数据范围为当前自然月，数据随任务状态变更实时更新。
- 巡检任务状态流转：待执行 → 进行中（首次签到后自动变更）→ 已完成（所有检查点完成后提交）；超过计划执行截止时间且未完成的任务，状态自动变更为已超期。
- GPS 签到有效范围为距检查点 50 米以内，超出范围时阻止签到，提示："当前位置距检查点超出有效范围，无法签到"。
- 必填检查项未全部填写时，不可提交当前检查点，提示："请完成所有必填检查项后再提交"。
- 系统每 30 秒自动记录一次当前用户的位置，用于生成巡检轨迹，记录范围为任务进行中的全程。
- 仅已完成的任务可查看完整轨迹回放；进行中的任务可查看实时轨迹；待执行和已超期任务无轨迹数据。

**边界与异常**

- **空数据**：各模块列表无数据时，分别显示对应的空数据提示文案（见各子功能说明）；轨迹无数据时，显示"暂无轨迹数据"。
- **网络异常**：数据加载失败时，显示"数据加载失败，请检查网络后重试"，支持手动刷新；检查清单提交失败时，保留已填写内容，提示："提交失败，请检查网络后重试"。
- **定位失败**：设备无法获取位置时，GPS 签到不可用，提示："无法获取当前位置，请检查定位权限后重试"。
- **NFC 不支持**：设备不支持 NFC 时，NFC 签到选项不展示。

**（三）业务逻辑关联**

- 待办事项中的巡检任务数据来源于巡检任务管理，需巡检任务管理中已完成任务分配。
- 待办事项中的整改工单数据来源于整改工单管理，需整改工单管理中已完成工单指派。
- 巡检路线及检查点数据来源于巡检路线管理，需巡检路线管理中已完成路线配置和检查点设置。
- 巡检任务执行中发起整改后，整改工单同步至整改工单管理，并在被指派人的待办事项中展示。
- 巡检任务状态变更同步至巡检任务管理，影响管理端的任务状态展示和统计数据。
- 通知公告数据来源于系统公告管理，需系统公告管理中已发布公告。
- 制度文档数据来源于制度文档管理，需制度文档管理中已发布文档。
- 统计图表数据随巡检任务状态变更同步更新，与巡检任务管理中的任务状态保持一致。

#### 3.5.10.2 工作台

**（一）功能需求概述**

工作台是移动端 App 的业务作业入口，集中提供发起整改、工单台账、设备管理、任务管理和巡检统计等现场作业功能，并按当前用户角色权限过滤可见入口，支持现场问题上报到整改跟踪的完整闭环。（巡检任务执行入口位于首页待办事项，点击巡检任务直接进入执行流程。）

**（二）功能设计**

工作台按业务功能划分为多个应用入口，各入口按角色权限展示。各入口功能说明如下。

##### 3.5.10.2.1 发起整改

**发起整改**

用户点击"发起整改"进入整改申请填写页面，拍照记录现场问题并填写相关信息后提交整改申请。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 现场照片 | 图片上传 | 1-10 张 | 是 | - | 至少上传 1 张，最多上传 10 张 |
| 问题描述 | 文本输入 | 最多 500 字 | 是 | - | 描述现场发现的问题 |
| 问题类型 | 单选 | - | 是 | - | 来源于问题类型配置中已启用的类型 |
| 关联巡检任务 | 只读文本 | - | 否 | 自动关联 | 系统自动关联当前执行中的巡检任务，不可手动修改 |

提交成功：显示"整改申请已提交"。
提交失败：保留填写内容，显示具体错误原因。

##### 3.5.10.2.2 工单台账

**工单台账**

展示与当前用户相关的整改工单记录列表，支持按状态和时间筛选，点击工单查看详情及完整流转历史。

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 工单状态 | 单选 | 枚举值：待审核 / 待整改 / 整改中 / 待复核 / 已完成 / 已关闭 |
| 时间范围 | 日期范围 | 选择开始和结束日期，按工单提交时间筛选 |

工单列表展示字段如下：

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 工单编号 | 文本 | 系统自动生成 |
| 问题描述 | 文本 | 截取前若干字，完整内容在详情页查看 |
| 问题类型 | 文本 | - |
| 工单状态 | 状态标签 | 待审核 / 待整改 / 整改中 / 待复核 / 已完成 / 已关闭 |
| 提交时间 | 日期时间 | - |

列表仅展示当前用户作为提交人、整改人或复核人的工单。列表为空时显示："暂无相关工单数据"。

点击工单条目，进入工单详情页，可查看工单基本信息及完整流转记录（含每个节点的操作人、操作时间和操作意见）。

##### 3.5.10.2.3 设备管理

**设备管理**

展示当前用户有权查看的设备档案列表，支持查看设备基本信息，并可对设备上报异常。

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 设备名称 | 文本 | - |
| 设备编号 | 文本 | - |
| 设备分类 | 文本 | - |
| 所在位置 | 文本 | - |
| 设备状态 | 状态标签 | 正常 / 停用 / 维修中 |

点击设备条目，进入设备详情页，可查看设备完整档案信息。详情页提供"上报异常"操作入口，点击后进入异常上报填写页面。列表为空时显示："暂无设备数据"。

##### 3.5.10.2.4 任务管理

**任务管理**

展示分配给当前用户的巡检任务列表及执行记录，用户可查看任务详情和历史执行情况。

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 任务名称 | 文本 | - |
| 巡检路线 | 文本 | - |
| 计划执行时间 | 日期时间 | - |
| 任务状态 | 状态标签 | 待执行 / 进行中 / 已完成 / 已超期 |

点击任务条目，进入任务详情页，可查看任务基本信息和历史执行记录。列表为空时显示："暂无任务数据"。

##### 3.5.10.2.5 巡检统计

**巡检统计**

以数据报表形式展示当前用户相关的巡检完成情况和异常统计数据，支持按时间范围查看。

| 统计指标 | 类型 | 说明 |
|---------|------|------|
| 巡检完成率 | 百分比 | 已完成任务数 / 计划任务总数 |
| 异常上报数 | 数值 | 统计周期内上报的异常总数 |
| 整改完成数 | 数值 | 统计周期内已完成整改的工单数 |

时间范围筛选支持按日、按周、按月切换查看。

**业务规则**

- 工作台各应用入口按当前用户角色权限过滤显示：巡检员可见发起整改、工单台账、设备管理、任务管理、巡检统计；整改人员可见工单台账；业务主管可见工单台账（审核）和设备管理；无相应权限的入口不展示。
- 发起整改时，若问题类型列表为空，阻止提交，提示："暂无可用问题类型，请联系管理员配置"。
- 发起整改时，现场照片未上传，阻止提交，提示："请至少上传 1 张现场照片"。
- 工单台账列表仅展示当前用户作为提交人、整改人或复核人的工单，其他用户的工单不在列表中显示。

**边界与异常**

- **空数据**：工单台账列表无数据时，显示"暂无相关工单数据"；设备列表无数据时，显示"暂无设备数据"；任务列表无数据时，显示"暂无任务数据"。
- **照片上传超限**：现场照片已达 10 张时，上传入口不可用，提示："最多上传 10 张照片"。
- **网络异常**：提交整改申请失败时，保留已填写内容，显示具体错误原因，不自动清空表单。

**（三）业务逻辑关联**

- 问题类型数据来源于后台问题类型配置，需管理员在后台完成配置后，发起整改功能中的问题类型选项方可正常使用。
- 发起整改提交后，系统生成整改工单，该工单将出现在工单台账列表中，并进入后台工单管理的审核流程。
- 设备管理中的设备档案数据来源于后台设备管理模块，移动端仅支持查看和上报异常，不支持新增或编辑设备档案。
- 巡检统计数据来源于任务管理和工单台账的汇总计算，数据范围与当前用户相关联。

#### 3.5.10.3 消息

**（二）功能设计**

**通知列表**

- 预警消息来源于巡检任务管理和整改工单管理的超期判定，与管理端的超期提醒一致。

#### 3.5.10.4 我的

**（一）功能需求概述**

"我的"页面为移动端用户提供个人信息查看、密码管理、系统设置和退出登录功能，用户可在此查看自身账号基本信息，并完成与账号安全和使用偏好相关的操作。

**（二）功能设计**

**个人信息**

展示当前登录用户的基本账号信息，所有字段均为只读，不支持在移动端直接编辑。

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 头像 | 图片 | 展示用户头像，无头像时显示默认占位图 |
| 姓名 | 文本 | 来源于用户管理中当前用户的姓名 |
| 所属部门 | 文本 | 来源于组织管理中当前用户所属的部门名称 |
| 角色 | 文本 | 来源于角色管理中当前用户被分配的角色名称 |

**修改密码**

用户点击"修改密码"进入密码修改页面，分步填写旧密码和新密码后提交。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 旧密码 | 密码输入 | - | 是 | - | 输入当前登录密码进行身份验证 |
| 新密码 | 密码输入 | 6-20 位 | 是 | - | 须同时包含字母和数字 |
| 确认新密码 | 密码输入 | 6-20 位 | 是 | - | 须与新密码完全一致 |

提交成功：显示"密码修改成功，请重新登录"，系统自动清除登录状态并跳转至登录页。
提交失败：保留填写内容，显示具体错误原因。

**系统设置**

提供消息通知和语言偏好的个性化配置，设置项即时生效。

| 设置项 | 类型 | 说明 |
|-------|------|------|
| 消息通知 | 开关 | 开启后接收系统推送通知；关闭后停止接收推送 |
| 语言设置 | 单选 | 选择 App 界面显示语言 |

**退出登录**

用户点击"退出登录"，弹出确认"确定要退出登录吗？"。确认后清除本地登录状态，跳转至登录页。取消后关闭确认提示，返回"我的"页面。

**业务规则**

- 旧密码验证失败时，阻止提交，提示："旧密码不正确，请重新输入"。
- 新密码格式不符合要求（不足 6 位、超过 20 位，或未同时包含字母和数字）时，阻止提交，提示："密码须为 6-20 位，且同时包含字母和数字"。
- 新密码与确认新密码内容不一致时，阻止提交，提示："两次输入的密码不一致，请重新输入"。
- 密码修改成功后，系统自动执行退出操作，用户须重新登录，不可停留在已登录状态。

**边界与异常**

- **空数据**：个人信息中若部门或角色未配置，对应字段显示"-"。
- **网络异常**：密码修改提交失败时，保留已填写内容，显示具体错误原因。

**（三）业务逻辑关联**

- 个人信息中的姓名、所属部门、角色数据来源于后台账号管理和组织管理模块，移动端仅展示，不支持在移动端修改。
- 密码修改成功后触发退出登录流程，与退出登录功能共用同一套登录状态清除机制。
- 消息通知开关影响系统推送通知的接收行为，与后台消息中心的推送配置联动。

# 四、技术方案

## 4.1 技术架构

**前端技术栈**

- PC管理后台：Vue 3 + TypeScript + Vite + Element Plus + Pinia，基于组件化开发，支持响应式布局
- 移动端：Vue 3 + Vant + Capacitor，适配主流移动浏览器，并支持打包为 Android / iOS 原生应用
- 数字大屏：Vue 3 + ECharts，支持全屏展示和自适应分辨率

**后端技术栈**

- 应用框架：NestJS，提供RESTful API服务
- 数据访问：Prisma ORM，统一管理数据模型与数据库迁移
- 权限框架：JWT + Passport，实现无状态身份认证和RBAC权限控制
- 任务调度：定时调度服务，实现巡检任务的定时自动生成
- 消息推送：WebSocket / 短信通知，实现实时消息推送

**数据库**

- 关系型数据库：MySQL，存储业务数据
- 缓存：Redis，缓存用户会话、权限数据和高频查询数据
- 文件存储：服务端本地存储（预留对象存储扩展），存储图片、附件等文件

**部署架构**

- 前端：Nginx静态资源服务器，支持HTTPS
- 后端：Docker容器化部署，支持水平扩展
- 数据库：主从复制，保障数据可用性
- 文件服务：独立文件存储服务

## 4.2 数据交互

**前后端交互方式**

- 采用RESTful API规范，HTTP/HTTPS协议
- 数据格式：JSON
- 统一响应格式：`{ code, message, data }`
- 分页数据格式：`{ total, pageNum, pageSize, list }`

**API设计原则**

- 资源命名使用名词，操作通过HTTP方法区分（GET/POST/PUT/DELETE）
- 接口版本管理：URL前缀 `/api/v1/`
- 统一错误码规范，前端根据错误码展示对应提示

**实时数据**

- 数字大屏数据：轮询（30秒间隔）
- 消息通知：WebSocket长连接，实现实时推送
- GPS轨迹上报：移动端每30秒上报一次位置

## 4.3 安全方案

**身份认证**

- 用户登录采用用户名+密码方式，密码加密存储（BCrypt）
- 登录成功后颁发JWT Token，Token有效期8小时
- 支持Token刷新机制，避免频繁重新登录

**权限控制**

- 基于RBAC模型，角色与菜单权限绑定
- 后端接口级别的权限校验，防止越权访问
- 数据权限控制：根据角色配置可查看的数据范围

**数据安全**

- 敏感数据（密码、手机号）加密存储
- 接口传输采用HTTPS加密
- 文件上传类型和大小限制，防止恶意文件上传
- SQL注入防护：使用参数化查询

**安全审计**

- 记录用户登录日志（时间、IP、设备）
- 记录关键操作日志（数据修改、权限变更）
- 日志保留180天

# 五、非功能需求

## 5.1 性能需求

- 页面加载时间：首屏加载 < 3秒，页面切换 < 1秒
- 接口响应时间：普通查询接口 < 500ms，复杂统计接口 < 3秒
- 并发用户数：支持100个并发用户同时在线
- 文件上传：单文件最大50MB，图片最大10MB

## 5.2 安全需求

- 身份认证：JWT Token认证，支持Token过期自动刷新
- 数据安全：敏感数据加密存储，传输采用HTTPS
- 权限控制：基于角色的访问控制（RBAC），接口级权限校验
- 防攻击：防SQL注入、XSS攻击、CSRF攻击

## 5.3 可用性需求

- 系统可用性：99.5%（每月允许停机时间 < 3.6小时）
- 故障恢复：RTO < 2小时，RPO < 1小时
- 数据备份：每日自动备份，备份数据保留30天
- 移动端离线：支持检查清单数据本地缓存，网络恢复后自动同步

## 5.4 兼容性需求

- PC端浏览器：Chrome、Firefox、Safari、Edge 最新版本
- 移动端：iOS 12+、Android 8.0+，支持微信内置浏览器
- 屏幕分辨率：PC端支持1920×1080、1366×768；移动端自适应
- 数字大屏：支持1920×1080、3840×2160（4K）分辨率
