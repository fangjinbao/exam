## 二、项目概述

### 2.1 项目背景

随着职业资格认证、企业技能培训、在线测评需求的持续增长，传统考试组织方式面临以下挑战：

1. **命题效率低、题库更新慢**：人工出题耗时耗力，题目质量参差不齐，难以快速扩充题库并保证覆盖各知识点。
2. **组卷依赖人工、难以兼顾均衡**：人工组卷需反复权衡题型、难度、知识点分布，效率低且不同卷面难度不一致。
3. **主观题阅卷成本高、标准不统一**：主观题依赖人工评卷，工作量大、周期长，且不同评卷人的评分尺度存在差异。
4. **在线考试作弊风险高**：远程考试缺乏有效的身份核验与行为监控手段，考试严肃性和公平性难以保障。
5. **多终端场景割裂**：考生在不同设备上的考试与练习体验不统一，缺乏一体化的平台支撑。

为解决上述问题，建设智能AI考试平台，引入人工智能辅助出题、组卷、阅卷，结合全链路防作弊技术，实现从命题到认证发证的一体化管理，提升考试组织效率与公平性。

### 2.2 项目建设目标

本项目旨在建设一套以人工智能为核心能力的在线考试与认证平台，实现以下目标：

1. **AI赋能命题与阅卷**：通过人工智能辅助批量出题、智能组卷、主观题自动阅卷，显著降低命题与评卷的人工成本，缩短考试组织周期。
2. **多终端一体化考试**：提供管理后台、监考端、网页考生端、桌面考试端、移动端多终端支持，满足不同场景下的考试与练习需求，数据统一管理。
3. **全链路防作弊保障**：集成防切屏检测、人脸核身、音视频与屏幕监控、操作限制、题目乱序等手段，配合实时监考，保障考试公平与严肃性。
4. **认证发证闭环管理**：支持资格认证项目管理、报考审核、考试通过后自动颁发可查验的电子证书，形成完整的认证业务闭环。
5. **数据驱动的学情分析**：基于考试与练习数据生成学情分析报告和统计报表，为教学与考核决策提供数据支撑。

### 2.3 项目建设范围

本项目建设范围包括：

**系统端**：

- 管理后台（Web应用）：面向管理与业务人员的全平台管理
- 监考端（Web应用）：面向监考人员的实时监控与异常处置
- 网页考生端（Web应用）：面向考生的在线考试与练习
- 桌面考试端（桌面应用）：面向考生的强防作弊考试
- 移动端（移动应用）：面向考生的移动考试与练习

**功能模块**：

- 题库管理：题目、题型、难度、知识点维护，AI出题与解析生成
- 试卷管理：固定试卷（手动组卷、AI组卷）与随机试卷（抽题规则）两种类型
- 考试管理：选择试卷创建考试、考生分配、防作弊策略配置
- 阅卷中心：客观题自动判分、主观题AI阅卷、人工复核
- 资格认证：认证项目、报考审核
- 监考中心：监考安排、异常处置、录像回放
- 证书管理：证书模板、颁发、查验
- 外部考生管理：外部单位考生信息维护与考试账号生成
- 报表中心：统计分析、学情分析
- 组织管理、用户管理、角色管理、系统管理

**数据范围**：

- 题库数据、试卷数据、考试与答卷数据、成绩与证书数据、认证与报考数据、考试行为与监考数据、系统配置数据

**不包含的功能**：

- 线下考场硬件对接（如闸机、专用摄像头）：预留接口，后续迭代实现
- 第三方支付与收费：本期不含考试收费相关功能

### 3.5 功能详细设计

### 3.5.1 题库管理

题库管理是命题的基础模块，维护题目及其分类体系，为组卷与练习提供题源。

#### 3.5.1.1 题目管理

**（一）功能需求概述**

题目管理用于维护考试系统的全部题目及其属性，支持人工录入、批量导入和 AI 辅助出题三种方式获取题源，并通过审核机制保障题目质量，为组卷与练习提供可靠的题目数据支撑。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 题干关键词 | 文本输入 | 支持模糊搜索 |
| 题型 | 单选 | 来源于题型管理中已启用的题型 |
| 难度 | 单选 | 来源于难度管理中已录入的难度等级 |
| 知识点 | 层级选择 | 来源于知识点分类中已维护的知识点 |

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 题干摘要 | 文本 | - |
| 题型 | 文本 | - |
| 难度 | 文本 | - |
| 所属知识点 | 文本 | - |
| 题目状态 | 状态标签 | 正式 / 待审 |
| 创建时间 | 日期时间 | - |
| 操作 | 操作按钮 | 编辑、删除（无权限时隐藏） |

分页默认每页10条，支持切换每页数量。列表为空时显示："暂无题目数据"。

**新增功能**

点击"新增题目"，弹出编辑窗口，填写题目信息。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 题干 | 多行文本 | - | 是 | - | - |
| 题型 | 单选 | - | 是 | - | 枚举值：单选题 / 多选题 / 判断题 / 填空题 / 问答题 / 论述题 |
| 选项 | 多行文本 | - | 客观题必填 | - | 仅单选题、多选题、判断题、填空题需填写 |
| 标准答案 | 文本输入 | - | 是 | - | - |
| 答案解析 | 多行文本 | - | 否 | - | - |
| 难度 | 单选 | - | 是 | - | 来源于难度管理中已录入的难度等级 |
| 知识点 | 层级选择 | - | 是 | - | 来源于知识点分类中已维护的知识点 |
| 分值建议 | 数字输入 | - | 否 | - | - |

提交成功：新增题目成功。
提交失败：保留填写内容，显示具体错误原因。

**编辑功能**

点击"编辑"，弹出编辑窗口，自动填入当前记录数据，表单字段同新增。
提交成功：编辑题目成功。

**删除功能**

点击"删除"，弹出确认"确定删除该题目吗？"。
删除成功：删除题目成功。
该题目已被固定试卷引用时，阻止删除，提示"该题目已被试卷引用，无法删除"。

**批量导入功能**

点击"批量导入"，下载标准模板，按模板填写后上传文件，确认导入。
导入成功：提示"成功导入 X 道题目"。

**AI出题功能**

点击"AI出题"，填写生成条件后提交，由AI自动生成题目草稿，生成的题目进入"待审"状态。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 知识点 | 层级选择 | - | 是 | - | 来源于知识点分类中已维护的知识点 |
| 题型 | 单选 | - | 是 | - | 来源于题型管理中已启用的题型 |
| 难度 | 单选 | - | 是 | - | 来源于难度管理中已录入的难度等级 |
| 生成数量 | 数字输入 | - | 是 | - | 不超过系统配置的单次AI出题最大数量 |

生成中：提示"正在生成，请稍候"。
生成成功：提示"已生成 X 道待审题目"，题目进入待审列表。
生成数量超出上限：阻止提交，提示"单次生成数量超出上限"。

**AI文档导入出题功能**

点击"AI文档导入出题"，上传资料文件后提交，由AI自动抽取知识点并生成题目草稿，生成的题目进入"待审"状态。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 资料文件 | 文件上传 | 单个≤20MB | 是 | - | 支持文档、表格等文本类资料 |

解析成功：提示"已从资料生成 X 道待审题目"。
解析失败：提示"资料解析失败，请检查文件内容或格式"。

**AI生成解析功能**

对指定题目点击"生成解析"，由AI自动生成该题目的答案解析。
生成成功：提示"解析生成成功"，解析自动填入该题目。
生成失败：提示"解析生成失败，请稍后重试"。

**待审题目审核功能**

在题目列表筛选出"待审"状态的题目，进入待审题目详情，逐题审核。

- 点击"审核通过"：题目转为"正式"状态，正式入库，提示"审核通过，题目已转为正式状态"。
- 点击"退回"：需填写退回原因（文本输入，必填，≤200字，说明退回理由），退回后题目保持"待审"状态并记录退回原因，提示"已退回"。

审核通过是AI生成题目进入正式题库的唯一入口。

**业务规则**

- AI生成的题目（含AI出题和AI文档导入出题方式生成的题目）必须经人工审核通过后才能进入正式题库，审核通过前状态保持"待审"。
- 题目被固定试卷引用时，删除需先解除关联，提示"该题目已被试卷引用，无法删除"。
- 客观题（单选题、多选题、判断题、填空题）新增/编辑时选项为必填项；主观题（问答题、论述题）无需填写选项。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无题目数据"。
- **权限不足**：新增、编辑、删除按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **AI服务不可用**：AI出题、AI文档导入出题、AI生成解析功能降级为人工录入，提示"AI服务暂不可用，请稍后重试或手动录入"。

**（三）业务逻辑关联**

- 题型数据来源于题型管理中已启用的题型，题型停用后AI出题与手动录入均不可选该题型。
- 难度数据来源于难度管理中已录入的难度等级。
- 知识点数据来源于知识点分类中已维护的知识点。
- 正式状态的题目被固定试卷组卷选用，并作为随机试卷抽题规则的抽取题源。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无分类数据"。
- **权限不足**：新增、编辑、删除按钮对无权限角色完全隐藏（不渲染，非禁用）。

**（三）业务逻辑关联**

- 知识点分类数据被题目管理引用，作为题目知识点字段和题目筛选的层级选择来源。

### 3.5.2 试卷管理

试卷管理用于将题库中的题目组织为可用于考试的试卷。试卷分为两种类型：**固定试卷**（题目固定，通过手动组卷或AI组卷生成）和**随机试卷**（配置抽题规则，考试时按规则为每位考生随机抽题）。两类试卷统一在同一列表中维护，供考试管理直接选用。

**（一）功能需求概述**

试卷管理用于维护固定试卷与随机试卷，支持按名称、类型、状态查询已有试卷，通过手动组卷、AI组卷生成固定试卷，或通过配置抽题规则生成随机试卷，为考试安排提供可直接引用的试卷数据。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 试卷名称 | 文本输入 | 支持模糊搜索 |
| 试卷类型 | 单选 | 枚举值：固定试卷/随机试卷 |
| 试卷状态 | 单选 | 枚举值：草稿/已发布 |

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 试卷名称 | 文本 | - |
| 试卷类型 | 状态标签 | 固定试卷/随机试卷 |
| 总分 | 数值 | 固定试卷为各题分值之和；随机试卷为抽题规则各项分值合计，系统自动计算 |
| 题目数量 | 数值 | 固定试卷为实际题数；随机试卷为抽题规则的总抽取数量 |
| 时长建议 | 数值 | 单位：分钟 |
| 试卷状态 | 状态标签 | 草稿/已发布 |
| 创建时间 | 时间 | 系统自动记录 |

| 时长建议 | 数值输入 | - | 是 | - | 单位：分钟 |
| 题目选择 | 多选 | 至少1题 | 是 | - | 来源于题库管理中已启用的题目 |
| 每题分值 | 数值输入 | - | 是 | - | 逐题设置，用于计算试卷总分 |

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 试卷名称 | 文本输入 | 2-50字 | 是 | - | - |

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|

| 难度 | 单选 | - | 是 | - | 来源于题库管理中已启用的难度等级 |
| 知识点 | 单选 | - | 是 | - | 来源于题库管理中已启用的知识点 |
| 抽取数量 | 数值输入 | - | 是 | - | 该题型/难度/知识点组合下抽取的题目数量 |
| 每题分值 | 数值输入 | - | 是 | - | 该组合下每道题的分值 |

提交失败：保留填写内容，显示具体错误原因。

**编辑功能**

点击"编辑"（仅草稿状态可用），自动填入当前试卷数据：固定试卷填入题目与分值，随机试卷填入抽题配置，表单字段同对应的新增功能。
提交成功：试卷修改成功。

**删除功能**

点击"删除"，弹出确认"确定删除该试卷吗？"。
删除成功：删除成功。
有关联数据时阻止删除，提示"该试卷已被考试引用，无法删除"。

**业务规则**

- 固定试卷总分等于各题分值之和；随机试卷总分等于抽题规则各项（抽取数量×每题分值）之和，保存前系统自动校验。固定试卷题目为空时阻止保存，提示"请至少添加一道题目"；随机试卷抽题配置为空时阻止保存，提示"请至少添加一条抽题配置"。
- 随机试卷某题型/难度/知识点组合下可用题目数量不足抽取数量时，保存或使用时提示"[题型/难度/知识点]可用题目不足，请调整抽取数量或补充题目"。
- 试卷被考试引用后不可删除，提示"该试卷已被考试引用，无法删除"。
- 试卷状态为已发布时，题目、分值与抽题配置不可编辑，编辑入口不可用。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无试卷数据"。
- **权限不足**：新增、编辑、删除按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **AI组卷服务不可用**：AI组卷服务暂不可用时，降级为手动组卷，提示"AI组卷服务暂不可用，请手动组卷"。
- **随机试卷可用题目不足**：保存或使用时校验各维度可用题目数量，不足时阻止提交，提示"[题型/难度/知识点]可用题目不足，请调整抽取数量或补充题目"。

**（三）业务逻辑关联**

- 固定试卷的题目、随机试卷抽题配置中的题型/难度/知识点，均来源于题库管理中已启用的数据
- 试卷被考试管理引用后，删除需先解除关联
- 随机试卷在考试时按抽题规则从题库中实时抽取题目，为每位考生生成不同卷面
- 试卷状态影响考试创建时可选试卷的范围

### 3.5.4 阅卷中心

**（一）功能需求概述**

阅卷中心用于对考试答卷进行评分。客观题在考生交卷后由系统自动判分，主观题由AI按评分标准打分并生成评语，阅卷老师可对AI评分结果进行复核与修正，复核过程留痕，最终汇总客观题与主观题得分形成总分并发布，供考生查询。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 考试名称 | 文本输入 | 支持模糊搜索 |
| 阅卷状态 | 单选 | 枚举值：待阅卷 / AI阅卷中 / 待复核 / 已完成 |
| 考生关键词 | 文本输入 | 支持模糊搜索 |

**阅卷任务列表**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 考试名称 | 文本 | - |
| 考生 | 文本 | - |
| 题型构成 | 文本 | 展示该答卷客观题、主观题的题量构成 |
| 客观题得分 | 数字 | 系统自动判分汇总，未交卷时显示"-" |
| 主观题得分 | 数字 | AI评分或人工复核后的汇总分，未完成阅卷时显示"-" |
| 总分 | 数字 | 系统自动计算：客观题得分 + 主观题得分，成绩未发布时显示"-" |
| 阅卷状态 | 状态标签 | 待阅卷 / AI阅卷中 / 待复核 / 已完成 |
| 操作 | 操作按钮 | 查看客观题判分、AI阅卷、人工复核、查看复核记录、发布成绩、撤回成绩（按状态和权限显隐） |

分页默认每页10条，支持切换每页数量。列表为空时显示："暂无阅卷任务"。

**客观题判分查看**

考生交卷后，系统即时按标准答案自动完成客观题判分，无需人工触发。查看时展示每道客观题的得分和对错结果。

| 题号 | 数字 | - |
| 考生答案 | 文本 | - |
| 标准答案 | 文本 | 数据来源：题库中该题配置的标准答案 |
| 得分 | 数字 | 系统自动判分 |
| 对错 | 状态标签 | 正确 / 错误 |

**主观题AI阅卷**

点击"AI阅卷"，系统对该答卷的主观题触发AI评分，点击后阅卷状态变更为"AI阅卷中"。AI按题目配置的评分标准逐题打分，并生成评语。

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 题号 | 数字 | - |
| 考生答案 | 文本 | - |
| 评分标准 | 文本 | 数据来源：题库中该题配置的评分标准 |
| AI评分 | 数字 | AI按评分标准自动打分 |
| AI评语 | 文本 | AI针对该题作答生成的评语 |

AI阅卷完成后，答卷阅卷状态变更为"待复核"。

**人工复核**

阅卷老师查看AI给出的每题评分与评语，可修改分数并填写复核意见。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| AI评分 | 只读 | - | - | AI阅卷结果自动带入 | 修改前分数 |
| 修改后分数 | 数字输入 | - | 是 | AI评分 | 阅卷老师复核后填写的最终分数 |
| 复核意见 | 多行文本 | ≤500字 | 是 | - | 阅卷老师填写的复核说明 |

提交成功：提示"复核完成，成绩已更新"，该答卷阅卷状态变更为"已完成"。
提交失败：保留填写内容，显示具体错误原因。

**复核记录查看**

查看某答卷的完整复核历史，记录不可删除、不可篡改。

| 字段名称 | 类型 | 说明 |
|---------|------|------|
| 复核人 | 文本 | 系统自动记录 |
| 复核时间 | 日期时间 | 系统自动记录 |
| 修改前分数 | 数字 | 系统自动记录 |
| 修改后分数 | 数字 | 系统自动记录 |
| 复核意见 | 文本 | 阅卷老师填写内容的留存 |

**成绩发布**

客观题得分与主观题最终得分汇总为总分。点击"发布成绩"，弹出确认"成绩发布后不可修改，确定要发布吗？"，确认后完成发布，考生可查询。
发布成功：提示"成绩发布成功"。

**成绩撤回**

对已发布成绩的答卷，点击"撤回成绩"，弹出确认"确定撤回该成绩吗？撤回后考生将无法查询"，确认后成绩撤回，阅卷状态回退为"待复核"，考生不可再查询该成绩。
撤回成功：提示"成绩已撤回"。

**业务规则**

- 客观题在考生交卷后由系统自动判分，无需人工干预。
- 主观题必须完成AI阅卷或人工评分后，该答卷才能发布成绩，未完成时阻止发布，提示"存在未完成阅卷的主观题，无法发布成绩"。
- 阅卷老师修改AI评分时，必须记录复核人、复核时间、修改前分数、修改后分数，复核记录不可删除。
- AI阅卷失败时，该主观题转为待人工评分状态，提示"AI阅卷失败，请人工评分"。
- 成绩发布后不可修改，如需修改需撤回成绩，提示"成绩已发布，如需修改请先撤回"。

**边界与异常**

- **空数据**：阅卷任务列表无数据时，显示"暂无阅卷任务"。
- **权限不足**：主观题AI阅卷、人工复核、成绩发布按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **AI服务不可用**：主观题AI阅卷降级为纯人工评分，提示"AI阅卷服务暂不可用，请人工评分"。

### 3.5.7 证书管理

#### 3.5.7.2 证书发放

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 认证项目 | 单选 | 来源于资格认证中已配置的认证项目 |
| 考生关键词 | 文本输入 | 支持按考生姓名模糊搜索 |
| 证书状态 | 单选 | 枚举值：有效 / 已作废 |

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 证书编号 | 文本 | 系统按模板编号规则自动生成，全局唯一 |
| 考生姓名 | 文本 | - |
| 认证项目 | 文本 | - |
| 颁发日期 | 日期 | 颁发时系统自动记录 |

| 操作 | 操作按钮 | 查看、下载、作废（无权限时隐藏） |

**颁发功能**

对通过考试且达到发证条件的考生，点击"颁发证书"，系统按对应证书模板的编号规则自动生成唯一证书编号并颁发。
颁发成功：证书颁发成功。
考生未达到发证条件时阻止颁发，提示"该考生未达到发证条件，无法颁发证书"。

**批量颁发功能**

选择某场考试，点击"批量颁发"，对该场考试中通过考试的考生批量颁发证书。
批量颁发完成后提示："批量颁发完成，成功X条，失败Y条"。

**作废功能**

点击"作废"，弹出确认窗口，填写作废原因后提交。
作废成功：证书已作废。
已作废证书不可恢复，作废前需再次确认"确定要作废该证书吗？作废后不可恢复"。

**查看/下载功能**

点击"查看"，展示证书完整内容。点击"下载"，将证书保存为文件。

- 只有考试通过（达到及格分数）且报考审核通过的考生才能被颁发证书。未达条件时阻止颁发，提示"该考生未达到发证条件，无法颁发证书"。
- 证书编号由系统按证书模板的编号规则自动生成，系统内全局唯一。
- 已作废证书不可恢复，作废操作须填写作废原因。
- 证书有效期由系统根据认证项目配置的有效期月数自动计算：有效期至 = 颁发日期 + 有效期月数。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无证书数据"。
- **权限不足**：颁发、批量颁发、作废按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **未达发证条件**：颁发时提示"该考生未达到发证条件，无法颁发证书"，阻止颁发。

**（三）业务逻辑关联**

- 证书发放依赖阅卷中心的考试成绩，只有达到及格分数的考生才具备发证资格。
- 证书发放依赖资格认证的报考审核结果，只有审核通过的考生才具备发证资格。
- 证书有效期依据认证项目中配置的有效期月数计算。
- 证书编号规则来源于证书模板管理。
- 考生颁发证书后，可在考生端"我的证书"中查看已获证书。

#### 3.5.7.3 证书查验

**（一）功能需求概述**

证书查验为社会公众提供公开的证书真伪核验入口，用户通过证书编号或查验码即可查询证书的真实性和有效状态，无需登录，提升证书的社会认可度。

**（二）功能设计**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|

**（三）业务逻辑关联**

- 查验结果数据来源于证书发放中已颁发的证书记录，证书状态实时同步。

### 3.5.9 报表中心

报表中心面向考试组织方和教学管理人员，提供考试与成绩的多维统计分析，并结合AI能力生成学情分析报告，为考试组织质量评估和教学决策提供数据支撑。报表中心下设考试统计、成绩分析、学情分析三个功能。

#### 3.5.9.1 考试统计

**（一）功能需求概述**

考试统计功能用于统计考试的整体组织情况，支持按考试、认证项目、时间范围查看场次、参考人数、成绩分布等核心指标，帮助管理人员掌握考试组织效果。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 考试名称 | 单选 | 来源于考试管理中已创建的考试 |
| 认证项目 | 单选 | 来源于资格认证中已启用的认证项目 |
| 时间范围 | 日期范围 | 选择开始和结束日期，按考试时间统计 |

**指标统计**

查看以下统计指标：

| 指标项 | 说明 |
|-------|------|
| 考试场次 | 统计范围内的考试场次总数；数据来源：考试管理与阅卷中心的成绩数据 |
| 参考人数 | 统计范围内实际参加考试的考生人数；数据来源：阅卷中心的成绩数据 |

| 最低分 | 统计范围内的最低成绩；数据来源：阅卷中心的成绩数据 |
| 通过率 | 统计范围内成绩达到合格标准的考生占比；数据来源：阅卷中心的成绩数据及考试合格标准 |

**趋势分析**

支持按时间维度查看考试场次与参考人数随时间的变化趋势，帮助识别考试组织的周期性规律。

**导出功能**

角色管理用于统一配置平台的角色及其功能访问权限，通过角色实现用户操作权限的批量管理，支持系统预置角色（系统管理员、考务人员、阅卷老师、监考员、考生）及权限的灵活调整，为全平台的权限控制提供基础支撑。

**（二）功能设计**

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 角色名称 | 文本 | - |
| 角色说明 | 文本 | 无值显示"-" |
| 关联用户数 | 数字 | 数据来源：系统自动统计当前已分配该角色的用户数量 |
| 是否内置 | 状态标签 | 是 / 否，数据来源：系统预置标记，不可修改 |
| 操作 | 操作按钮 | 编辑、分配权限、删除（内置角色不显示删除按钮，无权限时隐藏） |

分页默认每页10条，支持切换每页数量。列表为空时显示："暂无角色数据"。

**新增功能**

点击"新增角色"，弹出编辑窗口。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 角色名称 | 文本输入 | ≤50字 | 是 | - | 全局唯一 |
| 角色说明 | 文本输入 | ≤200字 | 否 | - | 角色用途说明 |

提交成功：新增角色成功。
提交失败：保留填写内容，显示具体错误原因。

**编辑功能**

点击"编辑"，弹出编辑窗口，自动填入当前角色数据，表单字段同新增。
提交成功：编辑角色成功。

**删除功能**

点击"删除"，弹出确认"确定要删除该角色吗？删除后不可恢复"。
删除成功：删除角色成功。
内置角色不支持删除，删除按钮不显示；若通过其他路径触发，提示"内置角色不可删除"。
角色下存在关联用户时阻止删除，提示"该角色下存在关联用户，无法删除，请先移除关联"。

**分配权限功能**

点击"分配权限"，进入该角色的权限配置页面，以菜单树形式逐项展示系统全部功能菜单，支持按菜单勾选可访问范围，并对每个菜单勾选具体操作权限（查看/新增/编辑/删除等），权限粒度到菜单与操作两级。
保存成功：权限配置已保存。

**业务规则**

- 角色名称在系统内全局唯一。新增/编辑时若名称重复，阻止提交，提示"该角色名称已存在，请更换"。
- 内置角色（系统管理员、考务人员、阅卷老师、监考员、考生）不可删除。点击删除时阻止操作，提示"内置角色不可删除"。
- 角色下存在关联用户时不可删除。删除时阻止操作，提示"该角色下存在关联用户，无法删除，请先移除关联"。
- 权限配置保存后，该角色下所有已关联用户的可访问功能范围实时更新，无需用户重新登录。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无角色数据"。
- **权限不足**：新增、编辑、删除、分配权限按钮对无权限角色完全隐藏（不渲染，非禁用）。

**（三）业务逻辑关联**

- 角色被用户管理引用，用户创建或编辑时需选择所属角色。
- 角色的权限配置结果决定页面访问权限矩阵中各角色对功能模块的访问范围。
- 分配权限页面展示的菜单数据来源于系统全部功能菜单。

| 连接状态 | 状态标签 | 正常 / 异常，由连接测试结果更新 |
| 操作 | 操作按钮 | 编辑、删除、启用/停用、连接测试（无权限时隐藏） |

分页默认每页10条，支持切换每页数量。列表为空时显示："暂无AI模型配置数据"。

**新增功能**

点击"新增"，弹出编辑窗口，填写模型配置信息。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |

| 模型服务商 | 单选 | - | 是 | - | 枚举值：通义千问 / DeepSeek / 智谱 / OpenAI / 其他 |
| 模型名称 | 文本输入 | ≤100字 | 是 | - | - |
| 接口地址 | 文本输入 | ≤200字 | 是 | - | 格式校验 |
| 密钥 | 文本输入 | ≤200字 | 是 | - | 加密保存，保存后脱敏展示，仅显示部分字符 |
| 最大并发数 | 数字输入 | - | 否 | - | 正整数 |
| 超时时间 | 数字输入 | - | 否 | - | 单位：秒 |

提交成功：显示"AI模型配置新增成功"。

**编辑功能**

点击"编辑"，弹出编辑窗口，自动填入当前记录数据，密钥字段展示脱敏值，表单字段同新增。
提交成功：显示"AI模型配置编辑成功"。

**删除功能**

点击"删除"，弹出确认"确定要删除该配置吗？删除后不可恢复"。
删除成功：显示"AI模型配置删除成功"。
该配置为当前启用模型时，阻止删除，提示"该配置为当前启用模型，无法删除"。

**切换启用功能**

点击"启用"，将该配置设为当前启用模型，原启用配置自动变更为停用状态。
切换成功：显示"已切换为当前启用模型"。

**连接测试功能**

点击"连接测试"，系统使用当前配置信息尝试连接对应模型服务，并将结果更新至连接状态列。
连接测试成功：连接状态更新为"正常"。
连接测试失败：连接状态更新为"异常"，提示"连接测试失败，请检查配置"。

- 同一时刻仅允许一个AI模型配置处于启用状态。切换启用某配置时，原启用配置自动变更为停用状态。

- **权限不足**：新增、编辑、删除、启用/停用、连接测试按钮对无权限角色完全隐藏（不渲染，非禁用）；系统管理菜单对非系统管理员角色隐藏。

### 3.5.14 考生端（网页/桌面）

考生端供考生参加考试、开展练习、管理错题、查询成绩与证书，网页端与桌面端共用同一套考生业务功能。桌面端在防作弊能力上更强，考试过程中提供强制全屏、切屏检测、快捷键屏蔽等网页端不具备的防作弊手段，其余功能与网页端一致。

#### 3.5.14.1 考试

**（一）功能需求概述**

考试功能供考生查看可参加的考试并完成在线作答，覆盖身份核验、答题、防作弊监控、断线续考与交卷的完整考试流程，为考试成绩的公正性提供保障。

**（二）功能设计**

**考试列表**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 考试名称 | 文本 | - |
| 开始时间 | 日期时间 | - |
| 时长 | 文本 | 以分钟展示 |
| 考试状态 | 状态标签 | 枚举值：未开始/进行中/已结束 |
| 是否已交卷 | 状态标签 | 枚举值：未交卷/已交卷 |

列表为空时显示："暂无考试"。

**身份核验**

进入考试前，按该考试配置的防作弊策略进行身份核验；若考试开启人脸核身，考生需完成人脸核验并通过后才能进入考试，核验未通过则不可进入。

**在线答题**

进入考试后逐题展示题目供考生作答，同时展示倒计时显示剩余考试时间；答案在作答过程中实时自动保存；支持在题目导航中查看已答与未答的标记，用于快速定位题目。

**防作弊（桌面端强化）**

- 考试过程中强制保持全屏状态。
- 检测切换到考试窗口以外的操作（切屏），记录切屏次数并进行告警；切屏次数超过该考试允许的上限时，系统自动强制交卷。
- 屏蔽复制、粘贴操作以及快捷键操作。
- 考试开启考中人脸抓拍时，按设定频率定时对考生进行人脸抓拍。

**断线续考**

考试过程中发生断线，重新连接后自动恢复到断线前的答题进度与剩余时间，已作答内容保持不丢失。

**交卷**

支持手动交卷，交卷前弹出确认"确定交卷吗？交卷后不可修改"；考试时间到时系统自动交卷，交卷后不可再修改答案。

**业务规则**

- 考试未到配置的开始时间时，考生不可进入考试，提示"考试尚未开始"。
- 考生已完成交卷后不可重复进入该考试，提示"您已完成本场考试"。
- 考试中切屏次数超过该考试允许的上限时，系统自动强制交卷，提示"切屏次数超限，系统已自动交卷"。
- 身份核验未通过时，考生不可进入考试，提示"身份核验未通过，无法进入考试"。

**边界与异常**

- **空数据**：考试列表无可参加的考试时，显示"暂无考试"。
- **断线**：考试过程中发生断线，已作答内容自动保存，不因断线丢失。
- **权限不足**：非分配给当前考生的考试不在列表中展示。

**（三）业务逻辑关联**

- 考试列表数据来源于考试管理中已发布且已分配给该考生的考试。
- 身份核验方式、切屏检测上限、是否开启人脸核身及考中抓拍频率均来源于该考试的防作弊策略配置。
- 考试中产生的切屏告警、强制交卷等作弊相关事件，上报至监考中心，并记录为异常事件。

### 3.5.15 监考端

监考端是独立的监考应用，供监考员在考试进行中实时监督分配给自己的考试，通过实时查看考生画面、接收异常告警并及时处置，保障考试过程的公平性和安全性。

#### 3.5.15.1 实时监控

**（一）功能需求概述**

实时监控功能供监考员集中查看分配给本人且正在进行的考试中全部在考考生的实时状态，支持查看单个考生详情、发送提醒并对确认违规的考生强制结束考试，保障监考过程及时有效。

**（二）功能设计**

**选择监考考试**

监考员进入后，展示分配给本人且状态为"进行中"的考试，来源于管理后台监考中心分配给该监考员的监考安排。监考员选择其中一场考试后进入该考试的监控。

**考生实时画面**

选择考试后，集中查看该考试全部在考考生的实时画面。

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 考生姓名 | 文本 | 数据来源：考生报名信息 |
| 实时画面 | 音视频画面、屏幕画面 | 来源于考生端开启音视频监控和屏幕监控的考试 |

**考生详情查看**

点击单个考生，查看该考生的实时画面、答题进度、已触发的异常。

| 字段名称 | 说明 |
|---------|------|
| 实时画面 | 音视频画面和屏幕画面 |
| 答题进度 | 数据来源：考生端答题记录，实时更新 |
| 已触发异常 | 数据来源：该考生的异常事件记录，按发生时间排列 |

点击"提醒"，填写提醒内容，向指定考生发送文字提醒。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|

**业务规则**

- 监考员仅能监控分配给自己且正在进行的考试。无进行中的监考任务时，提示"暂无进行中的监考任务"。
- 强制交卷为不可撤销操作，须二次确认"确定强制该考生交卷吗？操作不可撤销"。

**边界与异常**

- **空数据**：无分配给本人的进行中考试时，显示"暂无进行中的监考任务"。
- **权限不足**：远程提醒、强制交卷等操作对无权限角色完全隐藏（不渲染，非禁用）。
- **网络中断**：考生实时画面连接中断时，提示"画面连接中断，正在重连"。

**（三）业务逻辑关联**

- 监考任务来源于管理后台监考中心的监考安排，需监考中心先完成分配。
- 考生实时画面来源于考生端开启音视频监控和屏幕监控的考试，需考生端已开启相应监控。
- 强制交卷及提醒事件同步至监考中心的异常事件台账。

考生选择待参加的考试后进入作答。若考试开启人脸核身，需先完成身份核验方可进入。进入考试后在线作答，作答期间显示剩余时间倒计时，答案自动保存。作答完成后点击交卷，弹出确认"确定交卷吗？交卷后不可修改"，确认后完成交卷，交卷后不可再修改答案。作答期间系统检测切屏行为，切屏时记录告警。

**在线练习**

考生可按题库或知识点选择题目进行练习，作答后即时显示对错判断与解析内容。

**AI答疑**

考生可针对练习或考试中的题目向AI提问，获取解答内容。

**错题本**

考生可查看历次练习与考试中的错题，并对错题重新进行练习。

**拍照抓拍**

若考试配置要求进行身份留证，作答过程中按考试配置要求进行人脸抓拍并上传。

**业务规则**

- 考试未到设定的开始时间时，考生不可进入该考试，提示"考试尚未开始"。
- 考生已完成交卷的考试不可重复进入，提示"您已完成本场考试"。
- 考试开启人脸核身且核身未通过时，考生不可进入考试，提示"身份核验未通过，无法进入考试"。
- AI答疑服务不可用时，提示"AI答疑暂不可用，请稍后重试"。

**边界与异常**

- **空数据**：错题本无错题数据时，显示"暂无错题"。
- **权限不足**：未分配任何考试的考生，参加考试入口不展示可进入的考试。

**（三）业务逻辑关联**

- 参加考试的考试来源于考试管理中已发布且分配给该考生的考试。
- 在线练习的题目来源于题库管理。
- AI答疑功能依赖系统的AI模型配置。
- 考试过程中检测到的切屏等异常行为上报监考中心。

## 四、技术方案

### 4.1 技术架构

平台采用前后端分离架构，后端为单体模块化服务，对外提供统一接口，五个前端应用共用同一套业务数据。

**前端技术栈**：

- 管理后台、监考端、网页考生端：Vue 3 + TypeScript + Vite + Element Plus
- 桌面考试端：Electron 打包，具备强制全屏、切屏检测、快捷键屏蔽等强防作弊能力
- 移动端：Vue 3 + Capacitor，支持打包为移动应用

**后端技术栈**：

- 服务框架：NestJS（模块化组织，按业务模块划分）
- 数据访问：Prisma
- AI 能力：抽象为独立的 AI 服务模块，统一封装出题、组卷、阅卷、答疑、学情分析等能力，支持配置切换不同大模型服务商，密钥加密存储
- 实时监考：通过实时通信网关与缓存服务支撑监考端的画面查看与异常事件推送

**数据库与中间件**：

- 关系型数据库：MySQL，存储业务数据
- 缓存与队列：Redis，用于会话、实时消息、AI 长任务异步队列与削峰

**部署架构**：

- 各前端应用独立部署，后端服务与数据库、缓存分层部署
- AI 长任务（批量出题、整卷阅卷）通过异步队列执行，避免阻塞主服务

### 4.2 数据交互

- 前后端通过统一接口进行数据交互，采用标准数据格式，接口按业务模块划分
- 监考端的实时画面与异常告警通过实时通信方式传输
- AI 相关的长时任务采用异步处理，前端通过进度查询或实时推送获取处理进度
- 所有外部输入数据在服务端进行校验后再处理

### 4.3 外部系统集成

本期不涉及第三方业务系统集成。AI 大模型服务作为外部能力，通过 AI 抽象模块统一对接，支持按配置切换服务商。

### 4.4 安全方案

- **身份认证**：账号密码登录，登录态统一管理，密码加密存储
- **权限控制**：基于角色的访问控制（RBAC），接口与功能按角色权限校验，无权限操作对应角色完全隐藏
- **数据安全**：敏感数据（如AI模型密钥、用户密码）加密存储，传输采用加密协议
- **防作弊安全**：考试过程的身份核验、行为监控、录像留存，作弊事件全程留痕
- **审计**：关键操作记录操作日志，阅卷复核记录不可篡改删除

## 五、非功能需求

### 5.1 性能需求

- 页面加载时间 < 3 秒，常规接口响应时间 < 1 秒
- 支持考试高峰期的并发作答与提交，通过队列削峰保障批量交卷与批量阅卷的稳定
- AI 出题、整卷阅卷等长任务异步执行，不阻塞用户常规操作

### 5.2 安全需求

- 身份认证：账号密码登录，考试环节支持人脸核身
- 数据安全：敏感数据加密存储，传输使用加密协议
- 权限控制：基于角色的访问控制（RBAC）

### 5.3 可用性需求

- 系统可用性：99.9%
- 故障恢复：RTO < 1 小时，RPO < 15 分钟
- 数据备份：每日自动备份，考试与成绩数据重点保障
- 考试断线续考：考生断线重连后可恢复答题进度与剩余时间

### 5.4 兼容性需求

- 浏览器：Chrome、Firefox、Safari、Edge 最新版本
- 桌面端：Windows、macOS
- 移动端：主流 Android 与 iOS 设备
- 分辨率：桌面端 1920×1080、1366×768，移动端自适应
