## 二、项目概述

### 2.1 项目背景

随着职业资格认证、企业技能培训、在线测评需求的持续增长，传统考试组织方式面临以下挑战：

1. **命题效率低、题库更新慢**：人工出题耗时耗力，题目质量参差不齐，难以快速扩充题库并保证覆盖各知识点。
2. **组卷依赖人工、难以兼顾均衡**：人工组卷需反复权衡题型、难度、知识点分布，效率低且不同卷面难度不一致。
3. **主观题阅卷成本高、标准不统一**：主观题依赖人工评卷，工作量大、周期长，且不同评卷人的评分尺度存在差异。
4. **在线考试作弊风险高**：远程考试缺乏有效的身份核验与行为监控手段，考试严肃性和公平性难以保障。
5. **多终端场景割裂**：考生在不同设备上的考试与练习体验不统一，缺乏一体化的平台支撑。

为解决上述问题，建设智能AI考试平台，引入人工智能辅助出题、组卷、阅卷，结合全链路防作弊技术，实现从命题到认证发证的一体化管理，提升考试组织效率与公平性。

### 2.2 项目建设目标

本项目旨在建设一套以人工智能为核心能力的在线考试与认证平台，实现以下目标：

1. **AI赋能命题与阅卷**：通过人工智能辅助批量出题、智能组卷、主观题自动阅卷，显著降低命题与评卷的人工成本，缩短考试组织周期。
2. **多终端一体化考试**：提供管理后台、监考端、网页考生端、桌面考试端、移动端多终端支持，满足不同场景下的考试与练习需求，数据统一管理。
3. **全链路防作弊保障**：集成防切屏检测、人脸核身、音视频与屏幕监控、操作限制、题目乱序等手段，配合实时监考，保障考试公平与严肃性。
4. **认证发证闭环管理**：支持资格认证项目管理、报考审核、考试通过后自动颁发电子证书，形成完整的认证业务闭环。
5. **数据驱动的学情分析**：基于考试与练习数据生成学情分析报告和统计报表，为教学与考核决策提供数据支撑。

### 2.3 项目建设范围

本项目建设范围包括：

**系统端**：

- 管理后台（Web应用）：面向管理与业务人员的全平台管理
- 监考端（Web应用）：面向监考人员的实时监控与异常处置
- 网页考生端（Web应用）：面向考生的在线考试与练习
- 桌面考试端（桌面应用）：面向考生的强防作弊考试
- 移动端（移动应用）：面向考生的移动考试与练习

**功能模块**：

- 题库管理：题目、知识点分类维护，AI出题与解析生成（题型、难度由数据字典统一维护）
- 试卷管理：固定试卷（手动组卷、AI组卷）与随机试卷（抽题规则）两种类型
- 考试管理：选择试卷创建考试、考生分配、防作弊策略配置
- 阅卷中心：客观题自动判分、主观题AI阅卷、人工复核
- 资格认证：认证项目、报考审核
- 监考中心：监考安排、异常处置、录像回放
- 证书管理：证书模板、证书发放记录
- 外部考生管理：外部单位与外部考生信息维护、考试账号生成
- 报表中心：统计分析、学情分析
- 组织管理：部门管理、人员管理、岗位管理
- 考点管理、权限管理（角色管理、菜单管理）、系统管理

**数据范围**：

- 题库数据、试卷数据、考试与答卷数据、成绩与证书数据、认证与报考数据、考试行为与监考数据、系统配置数据

**不包含的功能**：

- 线下考场硬件对接（如闸机、专用摄像头）：预留接口，后续迭代实现
- 第三方支付与收费：本期不含考试收费相关功能

### 3.5 功能详细设计

### 3.5.1 题库管理

题库管理是命题的基础模块，按业务主题维护多个题库（如安全教育题库、信息安全管理题库），每个题库下包含若干题目，并维护题目所属的知识点分类体系，为组卷与练习提供题源。

#### 3.5.1.1 题库列表

**（一）功能需求概述**

题库列表用于维护平台的题库及其题目。**题库**是题目的容器，按业务主题划分（如"安全教育题库""信息安全管理题库"），每个题库下包含若干题目；管理员先在题库列表中维护题库，再进入指定题库维护其题目。题目支持人工录入、批量导入和 AI 辅助出题三种方式获取题源，并通过审核机制保障题目质量，为组卷与练习提供可靠的题库与题目数据支撑。

**（二）功能设计**

##### 一、题库列表

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 题库名称 | 文本输入 | 支持模糊搜索 |
| 状态 | 单选 | 启用 / 停用 |

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 题库名称 | 文本 | 如：安全教育题库、信息安全管理题库 |
| 题库编码 | 文本 | 题库唯一标识 |
| 题目数量 | 数值 | 该题库下题目总数，系统自动统计 |
| 状态 | 状态标签 | 启用 / 停用 |
| 创建时间 | 日期时间 | 系统自动记录 |
| 操作 | 操作按钮 | 进入题库、编辑、删除（无权限时隐藏） |

分页默认每页10条，支持切换每页数量。列表为空时显示："暂无题库数据"。

**新增题库功能**

点击"新增题库"，弹出编辑窗口，填写题库信息。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 题库名称 | 文本输入 | 2-50字 | 是 | - | 全局唯一 |
| 题库编码 | 文本输入 | ≤30字 | 否 | 系统生成 | 全局唯一，留空时系统自动生成 |
| 题库描述 | 多行文本 | ≤200字 | 否 | - | - |
| 状态 | 单选 | - | 是 | 启用 | 启用 / 停用 |

提交成功：新增题库成功。
提交失败：保留填写内容，显示具体错误原因。

**编辑题库功能**

点击"删除"，弹出确认"确定删除该题库吗？"。
删除成功：删除题库成功。
该题库下存在题目时，阻止删除，提示"该题库下存在题目，请先清空题目后再删除"。

**启用/停用**

题库停用后，组卷选题、随机试卷抽题、练习选择题库时均不展示该题库，历史已引用数据不受影响。

**AI生成解析功能**

对指定题目点击"生成解析"，由AI自动生成该题目的答案解析。
生成成功：提示"解析生成成功"，解析自动填入该题目。
生成失败：提示"解析生成失败，请稍后重试"。

**待审题目审核功能**

在题目列表筛选出"待审"状态的题目，进入待审题目详情，逐题审核。

- 点击"审核通过"：题目转为"正式"状态，正式入库，提示"审核通过，题目已转为正式状态"。
- 点击"退回"：需填写退回原因（文本输入，必填，≤200字，说明退回理由），退回后题目保持"待审"状态并记录退回原因，提示"已退回"。

审核通过是AI生成题目进入正式题库的唯一入口。

**业务规则**

- AI生成的题目（含AI出题和AI文档导入出题方式生成的题目）必须经人工审核通过后才能进入正式题库，审核通过前状态保持"待审"。
- 题目被固定试卷引用时，删除需先解除关联，提示"该题目已被试卷引用，无法删除"。
- 客观题（单选题、多选题、判断题、填空题）新增/编辑时选项为必填项；主观题（问答题、论述题）无需填写选项。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无题目数据"。
- **权限不足**：新增、编辑、删除按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **AI服务不可用**：AI出题、AI文档导入出题、AI生成解析功能降级为人工录入，提示"AI服务暂不可用，请稍后重试或手动录入"。

**（三）业务逻辑关联**

- 每道题目归属于唯一题库，题目的增删改、批量导入、AI出题、审核均在所属题库范围内进行。
- 题型数据来源于数据字典中"题型"已启用的字典项，字典项停用后AI出题与手动录入均不可选该题型。
- 难度数据来源于数据字典中"难度"已启用的字典项。
- 知识点数据来源于知识点分类中已维护的知识点。
- 正式状态的题目被固定试卷组卷选用，并作为随机试卷抽题规则的抽取题源；组卷与抽题均以题库为选题范围。
- 题库被试卷或练习引用后不可删除，题库下存在题目时不可删除。

### 3.5.2 试卷管理

试卷管理用于将题库中的题目组织为可用于考试的试卷。试卷分为两种类型：**固定试卷**（题目固定，通过手动组卷或AI组卷生成）和**随机试卷**（配置抽题规则，考试时按规则为每位考生随机抽题）。两类试卷统一在同一列表中维护，供考试管理直接选用。

**（一）功能需求概述**

试卷管理用于维护固定试卷与随机试卷，支持按名称、类型、状态查询已有试卷，通过手动组卷、AI组卷生成固定试卷，或通过配置抽题规则生成随机试卷，为考试安排提供可直接引用的试卷数据。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 试卷名称 | 文本输入 | 支持模糊搜索 |
| 试卷类型 | 单选 | 枚举值：固定试卷/随机试卷 |
| 试卷状态 | 单选 | 枚举值：草稿/已发布 |

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 试卷名称 | 文本 | - |
| 试卷类型 | 状态标签 | 固定试卷/随机试卷 |
| 总分 | 数值 | 固定试卷为各题分值之和；随机试卷为抽题规则各项分值合计，系统自动计算 |
| 题目数量 | 数值 | 固定试卷为实际题数；随机试卷为抽题规则的总抽取数量 |
| 时长建议 | 数值 | 单位：分钟 |
| 试卷状态 | 状态标签 | 草稿/已发布 |
| 创建时间 | 时间 | 系统自动记录 |
| 操作 | 操作按钮 | 预览、编辑（仅草稿状态可用）、删除（无权限时隐藏） |

分页默认每页10条，支持切换每页数量。列表为空时显示：暂无试卷数据。

**新增固定试卷（手动组卷）**

点击"手动组卷"进入新建试卷页面，先选择一个或多个题库，再从所选题库范围内挑选具体题目组合，逐题设置分值，并设置试卷名称与时长建议，生成固定试卷。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 试卷名称 | 文本输入 | 2-50字 | 是 | - | - |
| 时长建议 | 数值输入 | - | 是 | - | 单位：分钟 |
| 题库范围 | 多选 | 至少1个 | 是 | - | 来源于题库管理中已启用的题库 |
| 题目选择 | 多选 | 至少1题 | 是 | - | 来源于所选题库范围内已启用的正式题目 |
| 每题分值 | 数值输入 | - | 是 | - | 逐题设置，用于计算试卷总分 |

提交成功：试卷创建成功。
提交失败：保留填写内容，显示具体错误原因。

**新增固定试卷（AI组卷）**

点击"AI组卷"进入新建试卷页面，先选择组卷所用的题库范围，再输入总分、时长、知识点分布要求，由AI在所选题库范围内自动推荐一套组卷方案，可采纳或调整方案后保存为固定试卷。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 试卷名称 | 文本输入 | 2-50字 | 是 | - | - |
| 题库范围 | 多选 | 至少1个 | 是 | - | 来源于题库管理中已启用的题库，AI在此范围内选题 |
| 总分 | 数值输入 | - | 是 | - | AI按此总分生成推荐方案 |
| 时长建议 | 数值输入 | - | 是 | - | 单位：分钟 |
| 知识点分布要求 | 文本输入 | - | 否 | - | 描述各知识点占比或题量偏好 |

系统根据以上要求生成推荐题目方案，展示每道推荐题目及分值，支持对推荐方案进行调整（替换题目、调整分值）后保存。
提交成功：试卷创建成功。
提交失败：保留填写内容，显示具体错误原因。

**新增随机试卷（抽题规则）**

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 试卷名称 | 文本输入 | 2-50字 | 是 | - | - |
| 时长建议 | 数值输入 | - | 是 | - | 单位：分钟 |
| 题库范围 | 多选 | 至少1个 | 是 | - | 来源于题库管理中已启用的题库，抽题在此范围内进行 |
| 抽题配置 | 列表添加 | 至少1条 | 是 | - | 逐条设置题型、难度、知识点及对应抽取数量与分值 |

抽题配置字段表：

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 题型 | 单选 | - | 是 | - | 来源于数据字典中"题型"已启用的字典项 |
| 难度 | 单选 | - | 是 | - | 来源于数据字典中"难度"已启用的字典项 |
| 知识点 | 单选 | - | 是 | - | 来源于知识点分类中已维护的知识点 |
| 抽取数量 | 数值输入 | - | 是 | - | 该题型/难度/知识点组合下抽取的题目数量 |
| 每题分值 | 数值输入 | - | 是 | - | 该组合下每道题的分值 |

提交成功：试卷创建成功。
提交失败：保留填写内容，显示具体错误原因。

**预览功能**

点击"预览"，固定试卷查看完整题目与答案；随机试卷查看抽题配置与预计卷面构成。

**编辑功能**

点击"编辑"（仅草稿状态可用），自动填入当前试卷数据：固定试卷填入题目与分值，随机试卷填入抽题配置，表单字段同对应的新增功能。
提交成功：试卷修改成功。

**删除功能**

点击"删除"，弹出确认"确定删除该试卷吗？"。
删除成功：删除成功。
有关联数据时阻止删除，提示"该试卷已被考试引用，无法删除"。

**业务规则**

- 固定试卷总分等于各题分值之和；随机试卷总分等于抽题规则各项（抽取数量×每题分值）之和，保存前系统自动校验。固定试卷题目为空时阻止保存，提示"请至少添加一道题目"；随机试卷抽题配置为空时阻止保存，提示"请至少添加一条抽题配置"。
- 随机试卷某题型/难度/知识点组合下可用题目数量不足抽取数量时，保存或使用时提示"[题型/难度/知识点]可用题目不足，请调整抽取数量或补充题目"。
- 试卷被考试引用后不可删除，提示"该试卷已被考试引用，无法删除"。
- 试卷状态为已发布时，题目、分值与抽题配置不可编辑，编辑入口不可用。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无试卷数据"。
- **权限不足**：新增、编辑、删除按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **AI组卷服务不可用**：AI组卷服务暂不可用时，降级为手动组卷，提示"AI组卷服务暂不可用，请手动组卷"。
- **随机试卷可用题目不足**：保存或使用时校验各维度可用题目数量，不足时阻止提交，提示"[题型/难度/知识点]可用题目不足，请调整抽取数量或补充题目"。

**（三）业务逻辑关联**

- 固定试卷与随机试卷均先指定题库范围，固定试卷的题目、随机试卷抽题配置的题源均限定在所选题库范围内；题型/难度来源于数据字典，知识点来源于知识点分类
- 试卷被考试管理引用后，删除需先解除关联
- 随机试卷在考试时按抽题规则从所选题库范围内实时抽取题目，为每位考生生成不同卷面
- 试卷引用的题库不可被删除
- 试卷状态影响考试创建时可选试卷的范围

提交成功：创建考试成功。提交失败：保留填写内容，显示具体错误原因。

**编辑功能**

点击"编辑"，自动填入当前考试数据，表单字段同创建考试。仅未发布状态的考试可编辑。
提交成功：编辑考试成功。

**发布功能**

点击"发布"，考试状态变更为已发布，考生可见并可参加。

**撤回功能**

点击"撤回"，考试状态变更为未发布。仅未开始的考试可撤回。

**删除功能**

点击"删除"，弹出确认"确定删除该考试吗？"。
删除成功：删除考试成功。
已发布或进行中的考试不可删除，提示"考试已发布，无法删除，请先撤回"。

**业务规则**

- 结束时间必须晚于开始时间。保存时若不满足，阻止保存，提示"结束时间必须晚于开始时间"。
- 所选试卷为随机试卷时，若其抽题规则的可用题目数量不足，阻止发布，提示"随机试卷可用题目不足，无法发布考试"。
- 已发布或进行中的考试不可删除。删除时若命中该状态，阻止删除，提示"考试已发布，无法删除，请先撤回"。
- 考试到达开始时间后自动变更为进行中状态；到达结束时间后自动变更为已结束状态，并触发阅卷流程。
- 发布考试前必须已分配考生。发布时若未分配任何考生，阻止发布，提示"请先分配考生"。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无考试数据"
- **权限不足**：创建、编辑、删除按钮对无权限角色完全隐藏（不渲染，非禁用）
- **考生未分配**：发布时若未分配任何考生，阻止发布，提示"请先分配考生"

**（三）业务逻辑关联**

- 试卷数据（固定试卷与随机试卷）来源于试卷管理中已发布的试卷，需先在试卷管理中完成试卷维护
- 内部考生范围数据来源于组织管理的组织部门数据和组织管理·人员管理的内部考生数据；外部考生范围数据来源于外部考生管理，需先完成相应维护
- 关联认证项目数据来源于资格认证模块中已启用的认证项目
- 考试结束后，考试数据流向阅卷中心，触发阅卷流程

### 3.5.4 阅卷中心

**（一）功能需求概述**

阅卷中心用于对考试答卷进行评分。客观题在考生交卷后由系统自动判分，主观题由AI按评分标准打分并生成评语，阅卷老师可对AI评分结果进行复核与修正，复核过程留痕，最终汇总客观题与主观题得分形成总分并发布，供考生查询。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 考试名称 | 文本输入 | 支持模糊搜索 |
| 阅卷状态 | 单选 | 枚举值：待阅卷 / AI阅卷中 / 待复核 / 已完成 |
| 考生关键词 | 文本输入 | 支持模糊搜索 |

**阅卷任务列表**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 考试名称 | 文本 | - |
| 考生 | 文本 | - |
| 题型构成 | 文本 | 展示该答卷客观题、主观题的题量构成 |
| 客观题得分 | 数字 | 系统自动判分汇总，未交卷时显示"-" |
| 主观题得分 | 数字 | AI评分或人工复核后的汇总分，未完成阅卷时显示"-" |
| 总分 | 数字 | 系统自动计算：客观题得分 + 主观题得分，成绩未发布时显示"-" |
| 阅卷状态 | 状态标签 | 待阅卷 / AI阅卷中 / 待复核 / 已完成 |
| 操作 | 操作按钮 | 查看客观题判分、AI阅卷、人工复核、查看复核记录、发布成绩、撤回成绩（按状态和权限显隐） |

分页默认每页10条，支持切换每页数量。列表为空时显示："暂无阅卷任务"。

**客观题判分查看**

考生交卷后，系统即时按标准答案自动完成客观题判分，无需人工触发。查看时展示每道客观题的得分和对错结果。

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 题号 | 数字 | - |
| 考生答案 | 文本 | - |
| 标准答案 | 文本 | 数据来源：题库中该题配置的标准答案 |
| 得分 | 数字 | 系统自动判分 |
| 对错 | 状态标签 | 正确 / 错误 |

**主观题AI阅卷**

点击"AI阅卷"，系统对该答卷的主观题触发AI评分，点击后阅卷状态变更为"AI阅卷中"。AI按题目配置的评分标准逐题打分，并生成评语。

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 题号 | 数字 | - |
| 考生答案 | 文本 | - |
| 评分标准 | 文本 | 数据来源：题库中该题配置的评分标准 |
| AI评分 | 数字 | AI按评分标准自动打分 |
| AI评语 | 文本 | AI针对该题作答生成的评语 |

AI阅卷完成后，答卷阅卷状态变更为"待复核"。

**人工复核**

阅卷老师查看AI给出的每题评分与评语，可修改分数并填写复核意见。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| AI评分 | 只读 | - | - | AI阅卷结果自动带入 | 修改前分数 |
| 修改后分数 | 数字输入 | - | 是 | AI评分 | 阅卷老师复核后填写的最终分数 |
| 复核意见 | 多行文本 | ≤500字 | 是 | - | 阅卷老师填写的复核说明 |

提交成功：提示"复核完成，成绩已更新"，该答卷阅卷状态变更为"已完成"。
提交失败：保留填写内容，显示具体错误原因。

**复核记录查看**

查看某答卷的完整复核历史，记录不可删除、不可篡改。

| 字段名称 | 类型 | 说明 |
|---------|------|------|
| 复核人 | 文本 | 系统自动记录 |
| 复核时间 | 日期时间 | 系统自动记录 |
| 修改前分数 | 数字 | 系统自动记录 |
| 修改后分数 | 数字 | 系统自动记录 |
| 复核意见 | 文本 | 阅卷老师填写内容的留存 |

**成绩发布**

客观题得分与主观题最终得分汇总为总分。点击"发布成绩"，弹出确认"成绩发布后不可修改，确定要发布吗？"，确认后完成发布，考生可查询。
发布成功：提示"成绩发布成功"。

**成绩撤回**

对已发布成绩的答卷，点击"撤回成绩"，弹出确认"确定撤回该成绩吗？撤回后考生将无法查询"，确认后成绩撤回，阅卷状态回退为"待复核"，考生不可再查询该成绩。
撤回成功：提示"成绩已撤回"。

**业务规则**

- 客观题在考生交卷后由系统自动判分，无需人工干预。
- 主观题必须完成AI阅卷或人工评分后，该答卷才能发布成绩，未完成时阻止发布，提示"存在未完成阅卷的主观题，无法发布成绩"。
- 阅卷老师修改AI评分时，必须记录复核人、复核时间、修改前分数、修改后分数，复核记录不可删除。
- AI阅卷失败时，该主观题转为待人工评分状态，提示"AI阅卷失败，请人工评分"。
- 成绩发布后不可修改，如需修改需撤回成绩，提示"成绩已发布，如需修改请先撤回"。

**边界与异常**

- **空数据**：阅卷任务列表无数据时，显示"暂无阅卷任务"。
- **权限不足**：主观题AI阅卷、人工复核、成绩发布按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **AI服务不可用**：主观题AI阅卷降级为纯人工评分，提示"AI阅卷服务暂不可用，请人工评分"。

**（三）业务逻辑关联**

- 阅卷任务来源于考试管理中已结束的考试答卷。
- 主观题AI阅卷依赖AI模型配置（来源于系统管理中的AI模型配置）。
- 成绩发布后数据流向报表中心与证书管理，用于判断是否达到发证条件。

- 单位名称在系统内全局唯一。新增/编辑时若重复，阻止提交，提示"单位【XXX】已存在，请更换名称"。
- 单位编码填写时在系统内唯一。新增/编辑时若重复，阻止提交，提示"单位编码【XXX】已存在，请更换"。
- 联系电话填写时须符合11位手机号格式。
- 外部单位已被外部考生引用（存在归属于该单位的外部考生）时不可删除，单条删除与批量删除均校验此规则，阻止删除，提示"该单位已被外部考生引用，无法删除"。
- 外部单位停用后，外部考生新增/编辑时的所属单位选择范围中不再展示该单位；已关联该单位的历史外部考生数据不受影响。
- 导出按当前筛选条件全量导出，不受分页限制。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无外部单位数据"。
- **权限不足**：新增、编辑、删除、启用/停用按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **导出无数据**：当前筛选条件下没有可导出的数据时，提示"当前筛选条件下没有可导出的数据"。
- **导入空文件**：上传文件内容为空时，阻止导入，提示"文件内容为空，请按模板填写后再导入"。
- **导入多文件**：仅支持上传单个文件，多选时提示"仅支持上传单个文件，请先移除已选择的文件"。

**（三）业务逻辑关联**

- 外部单位停用后，外部考生录入时的所属单位可选范围自动排除该单位。
- 外部单位被外部考生引用时，删除需先解除关联（即先转移或清空相关外部考生的所属单位）。

| 手机号 | 文本输入 | 支持模糊搜索 |
| 账号状态 | 单选 | 枚举值：启用/停用 |

**列表功能**

| 列名 | 字段类型 | 说明 |

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |

| 证件号 | 文本输入 | ≤30字 | 是 | - | 用于身份核验的证件号码 |
| 密码 | 文本输入 | 6-20字 | 否 | 留空时取手机号后6位 | 仅新增时可填写，编辑时不展示 |

导入时逐行校验，跳过校验不通过的行并展示明细：

- 新增时密码留空，系统默认取手机号后6位作为初始密码。
- 重置密码固定将密码重置为该考生手机号后6位，非随机生成。
- 外部考生账号停用后，该考生无法登录考生端。

**边界与异常**

### 3.5.9 报表中心

报表中心面向考试组织方和教学管理人员，提供考试与成绩的多维统计分析，并结合AI能力生成学情分析报告，为考试组织质量评估和教学决策提供数据支撑。报表中心下设考试统计、成绩分析、学情分析三个功能。

#### 3.5.9.1 考试统计

**（一）功能需求概述**

考试统计功能用于统计考试的整体组织情况，支持按考试、认证项目、时间范围查看场次、参考人数、成绩分布等核心指标，帮助管理人员掌握考试组织效果。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 考试名称 | 单选 | 来源于考试管理中已创建的考试 |
| 认证项目 | 单选 | 来源于资格认证中已启用的认证项目 |
| 时间范围 | 日期范围 | 选择开始和结束日期，按考试时间统计 |

**指标统计**

查看以下统计指标：

| 指标项 | 说明 |
|-------|------|
| 考试场次 | 统计范围内的考试场次总数；数据来源：考试管理与阅卷中心的成绩数据 |
| 参考人数 | 统计范围内实际参加考试的考生人数；数据来源：阅卷中心的成绩数据 |
| 平均分 | 统计范围内所有考生成绩的平均值；数据来源：阅卷中心的成绩数据 |
| 最高分 | 统计范围内的最高成绩；数据来源：阅卷中心的成绩数据 |
| 最低分 | 统计范围内的最低成绩；数据来源：阅卷中心的成绩数据 |
| 通过率 | 统计范围内成绩达到合格标准的考生占比；数据来源：阅卷中心的成绩数据及考试合格标准 |

**趋势分析**

支持按时间维度查看考试场次与参考人数随时间的变化趋势，帮助识别考试组织的周期性规律。

**导出功能**

点击"导出"，将当前筛选条件下的统计数据导出为电子表格文件。

**业务规则**

- 通过率根据各考试设定的合格标准自动计算，不同考试的合格标准可能不同，统计时按各考试自身标准分别判定后汇总。
- 时间范围未选择时，默认统计全部历史考试数据。

**边界与异常**

- **空数据**：筛选结果无匹配数据时，显示"暂无统计数据"。
- **权限不足**：考试统计菜单入口对无权限角色完全隐藏（不渲染，非禁用）。
- **导出无数据**：当前筛选条件下无可导出数据时，提示"当前无可导出数据"。

**（三）业务逻辑关联**

- 考试场次、参考人数、平均分等指标数据来源于考试管理与阅卷中心的成绩数据，需相关考试完成阅卷后数据方可统计。
- 认证项目筛选数据来源于资格认证中已启用的认证项目。

#### 3.5.9.3 学情分析

**（一）功能需求概述**

学情分析功能基于考生的考试与练习数据，通过AI生成个人或群体的学情分析报告，识别薄弱知识点并给出能力评估与提升建议，为教学决策和个性化辅导提供依据。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 考生 | 单选 | 来源于组织管理·人员管理中已注册的考生；与组织部门二选一 |
| 组织部门 | 层级选择 | 来源于组织管理中已启用的部门，按层级展示；与考生二选一 |
| 时间范围 | 日期范围 | 选择开始和结束日期，按考试与练习发生时间统计 |

**AI学情报告**

系统调用AI，基于考生在统计范围内的考试与练习数据，生成学情分析报告。报告内容包含：

| 报告内容 | 说明 |
|---------|------|
| 薄弱知识点分析 | 识别考生在各知识点上的掌握薄弱环节 |
| 能力评估 | 评估考生当前的整体能力水平 |
| 提升建议 | 针对薄弱环节给出的学习提升建议 |

选择单个考生时生成个人学情报告；选择组织部门（班级/部门）时生成群体学情报告，反映该群体的整体学情。

**导出功能**

点击"导出"，将当前生成的学情分析报告导出为文件。

**业务规则**

- 生成学情报告需考生具备足够的考试或练习数据，数据不足时阻止生成，提示"该考生练习数据不足，暂无法生成学情分析"。
- AI服务不可用时阻止生成，提示"AI学情分析服务暂不可用，请稍后重试"。
- 群体学情报告基于群体内各考生的数据综合生成，反映群体整体学情，不替代个人学情报告。

**边界与异常**

- **空数据**：统计范围内无考试或练习数据时，显示"暂无统计数据"。
- **权限不足**：学情分析菜单入口对无权限角色完全隐藏（不渲染，非禁用）。
- **导出无数据**：当前无已生成的学情分析报告时，提示"当前无可导出数据"。
- **数据不足**：考生练习数据不足时，提示"该考生练习数据不足，暂无法生成学情分析"。
- **AI服务异常**：AI服务不可用时，提示"AI学情分析服务暂不可用，请稍后重试"。

**（三）业务逻辑关联**

- 学情分析依赖系统管理中的AI模型配置，需完成AI模型配置后方可生成报告。
- 学情分析所需的考试成绩数据来源于阅卷中心，练习数据来源于考生端的练习记录。
- 组织部门维度数据来源于组织管理中已启用的部门。

### 3.5.10 组织管理

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 部门名称 | 文本输入 | 支持模糊搜索，在已加载的部门数据中过滤 |

**列表功能**

按层级展示部门，支持逐级展开/收起下级部门。

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 部门名称 | 文本 | 按层级展示，支持逐级展开/收起下级部门 |
| 类型 | 状态标签 | 枚举值：省公司/分公司/部门 |
| 负责人 | 文本 | 未填写时留空展示 |
| 排序 | 数字 | - |
| 更新时间 | 日期时间 | 系统自动记录 |
| 操作 | 操作按钮 | 新增下级、编辑、删除（无权限时隐藏） |

| 上级部门 | 层级选择 | - | 否 | 顶级 | 来源于本模块中已有的部门，不选表示新增为顶级部门 |
| 部门类型 | 单选 | - | 是 | - | 枚举值：省公司/分公司/部门 |

支持多选，勾选后可进行批量删除。分页每页10/20/50/100条可切换。列表为空时呈现为空列表。

**新增功能**

点击"新增"，弹出编辑窗口，填写人员信息。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |

| 账号 | 文本输入 | - | 是 | - | 登录账号 |
| 密码 | 文本输入 | - | 是 | - | 仅新增时填写 |
| 手机号 | 文本输入 | ≤11字 | 否 | - | - |
| 所属部门 | 层级选择 | - | 是 | - | 来源于部门管理中已录入的部门 |
| 岗位 | 单选 | - | 否 | - | 来源于岗位管理中已录入的岗位 |
| 角色 | 多选 | - | 否 | - | 来源于角色管理中已配置的角色，可关联多个角色 |
| 邮箱 | 文本输入 | - | 否 | - | 填写时校验邮箱格式 |

提交成功：新增成功。
提交失败：保留填写内容，显示具体错误原因。
必填校验：姓名未填写时提示"请输入姓名"；账号未填写时提示"请输入账号"；所属部门未选择时提示"请选择部门"；邮箱格式不正确时提示"邮箱格式不正确"。

**编辑功能**

点击"编辑"，弹出编辑窗口，自动填入当前记录数据，表单字段同新增，不含密码字段，账号不可修改。
提交成功：更新成功。

**删除功能**

点击"删除"，弹出确认『确定要删除用户"{姓名或账号}"吗？』。
删除成功：删除成功。

**批量删除功能**

勾选多行后点击"批量删除"，弹出确认『确定要删除选中的 {N} 个用户吗？』。
删除成功：批量删除成功。
未勾选任何行时，批量删除按钮为禁用状态。

**状态切换功能**

点击状态开关，即时切换该人员的启用/禁用状态。
切换成功：状态更新成功。
切换失败：开关自动回滚为切换前状态。

**业务规则**

- 系统内置一个超级管理员账号，删除或批量删除时若选中包含该账号，整体拒绝操作，提示"超级管理员不允许删除"。
- 人员被禁用后不可登录，登录时提示"用户已被禁用"。
- 角色为多选，一个人员可同时关联多个角色。
- 账号在编辑时不可修改。
- 邮箱仅在填写时校验格式，格式不正确时提示"邮箱格式不正确"。

**边界与异常**

- **空数据**：列表无数据时，呈现为空列表。
- **未勾选批量操作**：未勾选任何行时，批量删除按钮禁用。
- **状态切换失败**：切换失败时自动回滚为切换前状态。
- **权限不足**：新增、编辑、删除、批量删除、状态切换按钮对无权限角色完全隐藏（不渲染，非禁用）。

**（三）业务逻辑关联**

- 所属部门数据来源于部门管理中已录入的部门。
- 岗位数据来源于岗位管理中已录入的岗位。
- 角色数据来源于权限管理·角色管理中已配置的角色，人员的角色决定其可访问的功能范围，与权限矩阵联动。
- 人员中的内部考生与外部考生管理中的外部考生，共同构成考试管理中考生分配的可选范围。

点击"分配权限"，弹出编辑窗口，以菜单树形式展示菜单管理中的全部菜单节点（目录、菜单、按钮三类节点同层级展示，含已设为隐藏的菜单），逐项带勾选框；打开时自动填入该角色当前已分配的菜单（自动勾选对应节点）。支持整体全选/取消全选（含部分选中状态），支持整体展开/折叠全部节点。

提交时，系统将已勾选节点与因其子节点被勾选而处于部分选中状态的上级节点一并保存，即勾选某个子节点时会自动为其上级目录一同建立访问权限。树中"按钮"类型节点的勾选即代表该角色获得对应的操作权限。

保存成功："分配权限成功"。
保存失败："分配权限失败"。
打开分配权限时获取菜单或已分配数据失败："获取权限数据失败"。

**业务规则**

- 角色标识为选填项，在系统内全局唯一。新增/编辑时若重复，阻止提交，按系统统一的重复数据处理机制提示"数据已存在（唯一约束冲突）"。
- 分配权限保存成功后，该角色下所有用户的可访问功能范围随之更新。
- 分配权限的菜单树包含已设为隐藏的菜单节点，隐藏菜单同样可被勾选纳入角色权限范围。

**边界与异常**

- **空数据**：列表无数据时，呈现为空列表。
- **权限不足**：编辑、分配权限、删除按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **取消删除**：取消操作不提示。

**（三）业务逻辑关联**

- 角色被人员管理中的用户角色字段引用；人员管理选择角色时不按角色状态过滤，即使角色已被禁用，仍可在人员管理中被选用。
- 分配权限页面展示的菜单数据来源于菜单管理中的全部菜单节点（含隐藏菜单）。
- 角色的权限配置结果决定该角色下用户可访问的功能范围。

#### 3.5.12.2 菜单管理

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，在已加载的菜单数据中过滤：

| 筛选字段 | 类型 | 说明 |

| 菜单类型 | 单选 | 枚举值：目录/菜单/按钮，在已加载的菜单数据中过滤 |

**列表功能**

| 列名 | 字段类型 | 说明 |

| 权限标识 | 文本输入 | - | 否 | - | 始终展示，如"sys:menu:list" |
| 排序 | 数字输入 | 0-9999 | 否 | 0 | 数值越小越靠前 |
| 是否显示 | 单选 | - | 是 | 显示 | 枚举值：显示/隐藏 |

提交失败：保留填写内容，提示"操作失败"。

提交成功："更新成功"。
提交失败：保留填写内容，提示"操作失败"。

切换失败："状态更新失败"，开关状态自动回退为切换前的值。

- 按钮类型节点只能挂在目录或菜单节点下。新增/编辑时上级菜单的可选范围排除按钮类型节点。
- 按钮类型节点承载具体操作的权限点，会与目录、菜单节点一同出现在角色管理·分配权限的菜单树中，供角色勾选。
- 按钮类型节点存在两种来源：管理员手动新增，或系统启动时按各功能模块声明的操作权限自动登记补全；列表中两种来源的节点不做区分标记。
- 菜单显示状态设为"隐藏"后，该菜单不在系统导航中展示，但仍完整出现在角色管理·分配权限的菜单树中，可被勾选纳入角色权限。

**边界与异常**

- **权限不足**：新增、编辑、删除按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **取消删除**：取消操作不提示。
- **显示切换失败**：开关状态自动回退为切换前的值，并提示"状态更新失败"。

**（三）业务逻辑关联**

- 菜单管理维护的全部菜单节点（含隐藏菜单）供角色管理·分配权限使用，作为角色可勾选的权限范围来源。
- 部分按钮类型节点由系统按各功能模块声明的操作权限自动登记补全，无需逐条手动录入。
- 上级菜单的选择范围来源于本模块中已有的目录与菜单节点，不含按钮节点。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |

| 密钥 | 密码输入 | ≤200字 | 是 | - | 加密保存，保存后脱敏展示，仅显示部分字符 |
| 模型名称 | 文本输入 | ≤100字 | 是 | - | 如：gpt-4o、claude-sonnet-4-20250514 |

提交成功：显示"AI模型配置新增成功"。
提交失败：保留填写内容，显示具体错误原因。

**编辑功能**

点击"编辑"，弹出编辑窗口，自动填入当前记录数据，密钥字段展示脱敏值且留空表示不修改密钥，表单字段同新增。
提交成功：显示"AI模型配置编辑成功"。

点击"删除"，弹出确认"确定要删除该配置吗？删除后不可恢复"。

**切换启用功能**

点击"启用"，将该配置设为当前启用模型，原启用配置自动变更为停用状态。
切换成功：显示"已切换为当前启用模型"。

**连接测试功能**

点击"连接测试"，系统使用当前配置信息尝试连接对应模型服务，并将结果更新至连接状态列。

**边界与异常**

### 3.5.14 考生端（网页/桌面）

#### 3.5.14.1 考试

**（一）功能需求概述**

考试功能供考生查看可参加的考试并完成在线作答，覆盖身份核验、答题、防作弊监控、断线续考与交卷的完整考试流程，为考试成绩的公正性提供保障。

**（二）功能设计**

**考试列表**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 考试名称 | 文本 | - |
| 开始时间 | 日期时间 | - |
| 时长 | 文本 | 以分钟展示 |
| 考试状态 | 状态标签 | 枚举值：未开始/进行中/已结束 |
| 是否已交卷 | 状态标签 | 枚举值：未交卷/已交卷 |

列表为空时显示："暂无考试"。

**身份核验**

进入考试前，按该考试配置的防作弊策略进行身份核验；若考试开启人脸核身，考生需完成人脸核验并通过后才能进入考试，核验未通过则不可进入。

**在线答题**

进入考试后逐题展示题目供考生作答，同时展示倒计时显示剩余考试时间；答案在作答过程中实时自动保存；支持在题目导航中查看已答与未答的标记，用于快速定位题目。

**防作弊（桌面端强化）**

- 考试过程中强制保持全屏状态。
- 检测切换到考试窗口以外的操作（切屏），记录切屏次数并进行告警；切屏次数超过该考试允许的上限时，系统自动强制交卷。
- 屏蔽复制、粘贴操作以及快捷键操作。
- 考试开启考中人脸抓拍时，按设定频率定时对考生进行人脸抓拍。

**断线续考**

考试过程中发生断线，重新连接后自动恢复到断线前的答题进度与剩余时间，已作答内容保持不丢失。

**交卷**

支持手动交卷，交卷前弹出确认"确定交卷吗？交卷后不可修改"；考试时间到时系统自动交卷，交卷后不可再修改答案。

**业务规则**

- 考试未到配置的开始时间时，考生不可进入考试，提示"考试尚未开始"。
- 考生已完成交卷后不可重复进入该考试，提示"您已完成本场考试"。
- 考试中切屏次数超过该考试允许的上限时，系统自动强制交卷，提示"切屏次数超限，系统已自动交卷"。
- 身份核验未通过时，考生不可进入考试，提示"身份核验未通过，无法进入考试"。

**边界与异常**

- **空数据**：考试列表无可参加的考试时，显示"暂无考试"。
- **断线**：考试过程中发生断线，已作答内容自动保存，不因断线丢失。
- **权限不足**：非分配给当前考生的考试不在列表中展示。

**（三）业务逻辑关联**

- 考试列表数据来源于考试管理中已发布且已分配给该考生的考试。
- 身份核验方式、切屏检测上限、是否开启人脸核身及考中抓拍频率均来源于该考试的防作弊策略配置。
- 考试中产生的切屏告警、强制交卷等作弊相关事件，上报至监考中心，并记录为异常事件。

## 四、技术方案

### 4.1 技术架构

平台采用前后端分离架构，后端为单体模块化服务，对外提供统一接口，五个前端应用共用同一套业务数据。

**前端技术栈**：

- 管理后台、监考端、网页考生端：Vue 3 + TypeScript + Vite + Element Plus
- 桌面考试端：Electron 打包，具备强制全屏、切屏检测、快捷键屏蔽等强防作弊能力
- 移动端：Vue 3 + Capacitor，支持打包为移动应用

**后端技术栈**：

- 服务框架：NestJS（模块化组织，按业务模块划分）
- 数据访问：Prisma
- AI 能力：抽象为独立的 AI 服务模块，统一封装出题、组卷、阅卷、答疑、学情分析等能力，支持配置切换不同大模型服务商，密钥加密存储
- 实时监考：通过实时通信网关与缓存服务支撑监考端的画面查看与异常事件推送

**数据库与中间件**：

- 关系型数据库：MySQL，存储业务数据
- 缓存与队列：Redis，用于会话、实时消息、AI 长任务异步队列与削峰

**部署架构**：

- 各前端应用独立部署，后端服务与数据库、缓存分层部署
- AI 长任务（批量出题、整卷阅卷）通过异步队列执行，避免阻塞主服务

### 4.2 数据交互

- 前后端通过统一接口进行数据交互，采用标准数据格式，接口按业务模块划分
- 监考端的实时画面与异常告警通过实时通信方式传输
- AI 相关的长时任务采用异步处理，前端通过进度查询或实时推送获取处理进度
- 所有外部输入数据在服务端进行校验后再处理

### 4.3 外部系统集成

本期不涉及第三方业务系统集成。AI 大模型服务作为外部能力，通过 AI 抽象模块统一对接，支持按配置切换服务商。

### 4.4 安全方案

- **身份认证**：账号密码登录，登录态统一管理，密码加密存储
- **权限控制**：基于角色的访问控制（RBAC），接口与功能按角色权限校验，无权限操作对应角色完全隐藏
- **数据安全**：敏感数据（如AI模型密钥、用户密码）加密存储，传输采用加密协议
- **防作弊安全**：考试过程的身份核验、行为监控、录像留存，作弊事件全程留痕
- **审计**：关键操作记录操作日志，阅卷复核记录不可篡改删除

## 五、非功能需求

### 5.1 性能需求

- 页面加载时间 < 3 秒，常规接口响应时间 < 1 秒
- 支持考试高峰期的并发作答与提交，通过队列削峰保障批量交卷与批量阅卷的稳定
- AI 出题、整卷阅卷等长任务异步执行，不阻塞用户常规操作

### 5.2 安全需求

- 身份认证：账号密码登录，考试环节支持人脸核身
- 数据安全：敏感数据加密存储，传输使用加密协议
- 权限控制：基于角色的访问控制（RBAC）

### 5.3 可用性需求

- 系统可用性：99.9%
- 故障恢复：RTO < 1 小时，RPO < 15 分钟
- 数据备份：每日自动备份，考试与成绩数据重点保障
- 考试断线续考：考生断线重连后可恢复答题进度与剩余时间

### 5.4 兼容性需求

- 浏览器：Chrome、Firefox、Safari、Edge 最新版本
- 桌面端：Windows、macOS
- 移动端：主流 Android 与 iOS 设备
- 分辨率：桌面端 1920×1080、1366×768，移动端自适应
